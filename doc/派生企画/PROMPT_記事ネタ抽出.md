# PROMPT_記事ネタ抽出

> [!note] 目的
> Obsidian などに溜めた雑多なメモを AI に読ませ、
> **GIRIN に関係する記事ネタを抽出・整理し、記事の構成案まで作る**ためのプロンプト案。

---

## 想定用途

- 日常メモや作業メモから、GIRINに関係する記事候補だけを抜き出したい
- 断片的な気づきを「記事になる主張」に変換したい
- どのネタが note 向きか、X向きか、保留かを仕分けしたい
- 将来的な note 運用自動化の下流入力として使いたい

---

## 使い方

1. Obsidian から関連メモをいくつか貼る
2. 下記プロンプトを AI に渡す
3. 抽出された記事候補の中から、使うものだけをプロジェクト側に転記する

---

## ベースプロンプト

```md
以下は、私が日常的に残している雑多なメモです。
GIRIN、2XKO、生成AI運用、個人開発、ゲーマーコミュニティ運営に関係する話題が混ざっています。

あなたの仕事は、これらのメモをそのまま要約することではありません。
「あとで記事にできるネタ」を抽出し、記事候補として整理することです。

次のルールで処理してください。

【目的】
- メモから記事化できるテーマを抽出する
- GIRINと接続できるものを優先する
- 単なる感想ではなく、読者に渡せる主張や学びに変換する

【抽出基準】
- 実体験がある
- 判断の変化がある
- 他人に共有すると役立つ学びがある
- GIRIN / 2XKO / 生成AI / 個人開発 / コミュニティ運営のいずれかに接続する

【除外基準】
- 単なる感情ログ
- 具体性がなく記事の軸にならないもの
- GIRINとの接点が薄すぎるもの
- 事実関係が弱く、体験ベースでも論点ベースでもないもの

【出力してほしい内容】
1. 有望な記事候補を3〜10件
2. 各候補について以下を出す
   - 仮タイトル
   - 分類（AI運用 / コミュニティ運営 / 2XKO初心者向け / GIRIN周辺 / その他）
   - なぜ記事になるか
   - この記事の主張
   - 想定読者
   - 使えそうな具体例
   - 不足している材料
   - 優先度（高 / 中 / 低）
3. 候補に入れなかったが、将来育つ可能性があるメモがあれば「保留ネタ」として別に出す

【重要】
- 元メモの言い換えだけで終わらず、読者向けの論点に変換する
- 大げさに一般化しすぎない
- 私が実際に経験していないことは、経験したように書かない
- 不明な点は「不足している材料」として扱う
- 最後に「今すぐ記事化するならどれか」を上位3件だけ選んで理由を書く

以下にメモを貼ります。
```

## 構成案まで出したい時の追加プロンプト

```md
上で抽出した記事候補のうち、優先度が高いものについて、さらに記事構成案を作ってください。

各候補について以下を出してください。
- 仮タイトル
- 一番伝えたいことを1文
- 導入で使う問題提起
- 本文の見出し案を3〜5個
- 各見出しで何を書くか
- 具体例として使うべき手元の体験
- 読後に読者が持ち帰るもの
- GIRINへの自然な接続がある場合はその導線

注意:
- note の記事として自然な構成にする
- AIっぽい総花的な見出しにしない
- 私の体験が芯になるようにする
```

---

## 1本だけ深掘りしたい時の追加プロンプト

```md
以下の記事候補を1本だけ深掘りしてください。

【対象】
- 仮タイトル: （ここに入れる）

次の内容を出してください。
- このテーマで書く価値
- 想定読者の具体像
- この記事で避けるべきズレた方向
- 記事の骨子
- 冒頭の書き出し案を3つ
- 見出し案
- 使える具体例
- 不足している材料
- このまま下書きに進めるかどうかの判断
```

---

## 出力の見方

- `優先度: 高`
  実体験があり、主張も明確で、そのまま記事案にしやすい
- `優先度: 中`
  軸はあるが、具体例か結論が足りない
- `優先度: 低`
  断片としては面白いが、単独記事には弱い
- `保留ネタ`
  今は弱いが、今後の体験や事例が増えると化ける

---

## メモ

- 雑メモを一度に大量投入するより、テーマが近いメモを10〜30件ずつ投げた方が精度が上がりやすい
- AIに最初から本文を書かせるより、先に「候補抽出」と「構成化」を分けた方が扱いやすい
- GIRINと無関係な日常メモまで混ぜる場合は、貼る前に `#girin` `#記事ネタ` のようなタグで絞るとよい

# コンボタイムライン動画

> [!note] 位置づけ
> 本筋（GIRIN サービス開発）に接続する**派生企画**。
> 短尺コンボ動画とは別に、コンボの押すべきボタン・タイミング・失敗原因をわかりやすく伝える教材型動画を検討する。

---

## 概要

格闘ゲームのコンボ動画に、音ゲーノーツのような入力ガイドを重ねる。

ただし、目的は「音ゲー風に見せること」ではない。
GIRINの核心である **「これだけ見れば練習できる安心感」** を補強するために、次に押すボタン、押してよい受付、失敗時の直し方を見えるようにする。

呼び方は、現時点では **コンボタイムライン** がよい。

- 音ゲーノーツ式よりも格ゲー向けの説明に寄せられる
- フレーム、受付、注釈、失敗診断まで含めやすい
- GIRINの入力データやWeb UIと接続しやすい

---

## 背景

過去にも、格ゲーのコンボ入力を音ゲーノーツのように流す先行事例はあった。
しかし広く定着したとは言いにくい。

主な理由は、格闘ゲームのコンボ学習と音ゲーUIの得意領域が完全には一致しないため。

| 論点 | 内容 |
|---|---|
| 格ゲーの入力は拍ではなく状況依存 | ヒットストップ、キャンセル受付、相手の浮き、着地、端距離、アシストなどに左右される |
| 情報量が多い | 方向、ボタン、ジャンプ、ディレイ、ホールド、同時押し、タッグ、アシストを単純なレーンに落とし込みにくい |
| 視線が分散する | ゲーム画面、相手の高さ、入力、字幕、ダメージを同時に見る必要がある |
| 短尺動画と教材動画の目的が違う | ショート動画は「見て気持ちいい」が強く、教材は「止めて練習できる」が重要 |
| 制作コストが高い | 手作業でノーツや注釈を合わせると、1本ごとの負担が重い |

そのため、単にボタンを流すだけでは、見た目はわかりやすくても練習教材としては弱くなりやすい。

---

## 重要視するべき観点

### 1. 「いつ押すか」を1点ではなく受付で見せる

音ゲーは判定ラインに来た瞬間に押す。
一方、格ゲーのコンボは、キャンセル受付や目押し猶予の中に入力を入れることが多い。

そのため、ノーツは「この1フレームで押せ」という表現ではなく、**押してよい時間帯を見せる**方がよい。

```text
次: H
      ┌──── キャンセル受付 ────┐
──────████████████████████──────
              ▲ 推奨入力
```

これにより、初心者が「完全にジャストで押さないといけない」と誤解しにくくなる。

### 2. フレームメーターやキャンセル表示と組み合わせる

現代の格ゲーには、コンボ学習を助ける可視化機能が増えている。

- SF6: キャンセル可能タイミングでキャラの色が変わる表示
- 2XKO: トレーニングモードにフレームメーター系の可視化がある

これらとコンボタイムラインを組み合わせると、単なる入力順の表示ではなく、**なぜそのタイミングで押すのか** を説明できる。

役割分担は以下。

| 要素 | 役割 |
|---|---|
| ノーツ表示 | 次に押すボタンを先読みさせる |
| フレームメーター | どの受付・有利状況で押せるのかを見せる |
| ゲーム画面 | 相手の高さ、着地、距離、アシストの当たり方を見る |
| 注釈 | 早い、遅い、少し待つ、何ヒット目、などを言語化する |

### 3. 全入力を均等に説明しない

すべての入力を同じ密度で解説すると、画面がうるさくなる。
価値が出るのは、初心者が失敗しやすい場所だけを厚く説明すること。

重点的に可視化するべき箇所:

- `dl.` が入るディレイ入力
- 多段技の何ヒット目キャンセル
- ジャンプ、着地、ダッシュ後の拾い
- アシスト呼び出しのタイミング
- タッグ後の残キャラ入力
- 相手の高さで成功/失敗が分かれる部分

逆に、`M > H > S1` のような単純な連係は、入力列として見せるだけでもよい。

### 4. 失敗診断を入れる

初心者にとって重要なのは、正解入力だけではない。
失敗したときに、早いのか遅いのか、距離が足りないのかを判断できること。

| 症状 | 伝えるべきこと |
|---|---|
| 相手が高すぎて次が当たらない | 入力が早い可能性 |
| 相手が落ちて受け身される | 入力が遅い可能性 |
| 必殺技が出ない | キャンセル受付前に押している可能性 |
| 次の技が空振る | ダッシュ距離や前進量が足りない可能性 |
| アシストが当たらない | 呼び出しが早い/遅い可能性 |

これはGIRINの「対戦に送り出す」思想と相性がよい。
ただコンボを眺める動画ではなく、練習中に詰まった場所を直せる動画になる。

### 5. チャンク化する

長いコンボを1本の譜面として流すと、初心者には追いにくい。
3〜5個の小さな塊に分ける。

例:

```text
1. 始動
M > H > S1

2. 浮かせ直し
2S1 > dl.M > j.2H

3. アシスト拾い
j.S2 > 6T:JINX > j.2H

4. 締め
2T:JINX > j.H > j.2H > S1+S2
```

動画上でも、チャンクごとに短い見出しを出す。

## 動画構成案

### 基本構成

```text
1. 完成形を通常速度で見せる
2. スロー再生でコンボタイムラインを表示
3. 難所だけフレームメーター/色/相手の高さを注釈
4. 失敗診断を短く入れる
5. 最後に入力列を固定表示
```

### 9:16ショート向け

- 冒頭1〜2秒で完成形の見どころを出す
- ノーツは画面下部に置く
- 重要入力だけ大きく表示
- 字幕は短く、1画面1メッセージにする
- 最後に「GIRINで入力確認」の導線を置く

### GIRIN内の教材向け

- ショート動画より情報量を増やしてよい
- 入力列、チャンク、失敗診断を固定表示できる
- 一時停止して見返す前提にする
- コンボカードの `inputs` と同じ表記を使う

## 制作コストの問題

この企画の最大のボトルネックは制作コスト。

手作業でやると、以下を毎回作る必要がある。

- コンボ録画
- トリミング
- ノーツのタイミング合わせ
- 譜面デザイン
- 解説台本
- 字幕
- ナレーション
- 最終書き出し

そのため、最初から全コンボ対応を目指さない。
まずはおすすめコンボ、初心者向けコンボ、失敗しやすいコンボだけで試す。

## AI動画編集の考え方

生成AIに動画そのものを丸投げするのは、この用途では危険。

理由:

- 変な文字が出る
- 存在しないUIが出る
- ボタン表記が間違う
- タイミングがズレる
- ゲーム画面が実録と違うものになる

向いているのは、**実録素材 + 正しいデータ + テンプレート合成**。

```text
実録コンボ動画
+ 正しい入力データ
+ ノーツのタイミング表
+ 解説台本
+ デザインテンプレート
→ 動画生成API / プログラムで合成
→ AIは台本・字幕・ナレーション・ラフ編集を担当
```

AIに任せてよい領域:

- 解説台本の初稿
- 字幕文の整形
- ナレーション原稿
- SNS向けタイトル案
- 失敗診断の文章化
- ラフな尺調整案

AIに任せすぎない領域:

- ボタン表記の正確性
- ノーツのタイミング
- フレームメーターの読み取り
- 実際にコンボが繋がるかどうか
- ゲーム画面そのものの生成

## おすすめ構成

### MVP

まずは1本だけ、完全自動化ではなくテンプレ検証として作る。

```text
1. コンボ動画を録画
2. ffmpeg / 既存ツールで切り出し
3. 入力列をGIRIN形式で用意
4. 手入力でタイミング表を作る
5. Remotionでノーツと注釈を合成
6. AIで台本・字幕案を作る
7. 人間が最終確認して書き出し
```

タイミング表のイメージ:

```json
[
  { "frame": 12, "input": "M", "note": "" },
  { "frame": 28, "input": "H", "note": "キャンセル" },
  { "frame": 44, "input": "S1", "note": "光ったら押す" },
  { "frame": 78, "input": "dl.M", "note": "少し待つ" }
]
```

### 少量運用

```text
Remotionローカル
+ AI台本生成
+ VOICEVOXナレーション
+ 必要ならDescript/Kapwingを1ヶ月だけ
```

月額をほぼゼロにしつつ、字幕やナレーションだけ省力化する。

### 量産運用

```text
GIRINのコンボデータ
+ 入力タイミングデータ
+ テンプレート
→ Shotstack / JSON2Video / Creatomate でクラウド書き出し
```

月に何十本も作る段階で検討する。
初期から入れると、テンプレが固まる前に月額コストだけ発生しやすい。

## GIRINとの接続

GIRINにはすでに `inputs` フィールドがある。
この入力表記を土台にすれば、コンボタイムラインの自動生成に発展できる。

将来的なデータ追加候補:

| フィールド | 用途 |
|---|---|
| `timeline` | 入力ごとのフレーム/秒数 |
| `input_notes` | 難所の注釈 |
| `practice_tips` | 失敗診断 |
| `video_slow_url` | スロー解説動画 |
| `video_timeline_url` | コンボタイムライン動画 |

ただし、Phase 1ではDBやデータ構造を増やしすぎない。
まずは外部ファイルや試作JSONで検証する。

## 導入優先度

最初に作るなら、以下のコンボを優先する。

- 初心者向けおすすめコンボ
- GIRIN内で一番見られるコンボ
- `dl.` が入るコンボ
- アシスト/タッグでタイミングがわかりにくいコンボ
- 「押しているのに繋がらない」が起きやすいコンボ

逆に、簡単なコンボや見ればすぐ真似できるコンボは後回しでよい。

## メモ

- この企画は「短尺コンボ動画の代替」ではなく「練習用の別フォーマット」として考える
- 制作コストが高いため、全コンボ対応を最初から目指さない
- AIは動画そのものを生成する役ではなく、台本・字幕・ナレーション・説明文の補助役にする
- 正確性が教材としての価値なので、最終確認は必ず人間が行う

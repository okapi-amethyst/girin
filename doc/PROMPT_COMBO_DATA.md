# 追加プロンプト: コンボデータ設計

> PROJECT_PROMPT.md と合わせて使用

## キャラクターマスターデータ

キャラクター情報を1箇所で管理し、コンボデータからはIDで参照する。

| フィールド | 型     | 説明             |
| ------ | ------ | ---------------- |
| id     | number | 一意ID（1始まりの連番）。DB化前の仮ID |
| slug   | string | URL用スラッグ（例: `yasuo`, `ahri`） |
| name   | string | 英語表示名（例: `Yasuo`） |
| nameJa | string | 日本語表示名（例: `ヤスオ`） |

## コンボ1枚のデータ構造

### コンボ基本情報

| フィールド            | 型      | 必須  | 説明                                 |
| ---------------- | ------ | --- | ---------------------------------- |
| id               | number | ✅   | 一意ID（1始まりの連番）。DB化前の仮ID。パルス相互参照等に使用 |
| point_character  | number | ✅   | ポイントキャラクターID（キャラクターマスターの `id` を参照） |
| assist_character | number |     | アシストキャラクターID。ソロコンボの場合は `null`      |
| game_version     | string |     | 対応ゲームバージョン/パッチ（例: "1.2.0"）。Phase 2以降はDB側の `created_at` で管理。Phase 1では省略可 |

### コンボ条件

| フィールド             | 型       | 必須  | 説明                                                                                                          |
| ----------------- | ------- | --- | ----------------------------------------------------------------------------------------------------------- |
| fuse              | string  | ✅   | 使用Fuse名。不要なら `"No Fuse"`                                                                                    |
| starter           | enum    | ✅   | 始動状況。選択肢: `ground`（対地上） / `anti_air`（対空） / `throw`（投げ） / `pressure_mixup`（固め・崩し） / `counter_hit`（カウンターヒット） |
| position          | enum    | ✅   | 位置: `center` / `near_corner` / `corner` / `near_corner_back` / `corner_back`                                |
| duo_only          | boolean | ✅   | デュオ限定のコンボか。デフォルト `false`                                                                                    |
| ~~tag~~           | —       | —   | リソース消費セクションに移動。`inputs` から自動導出 |
| pulse             | boolean | ✅   | パルス使用コンボか。デフォルト `false`                                                                                     |
| related_combo_id  | number  |     | 同ルートのパルス有/無バージョンのコンボID（相互参照）                                                                                   |
| target_characters | object  | ✅   | `{ "type": "all" }` または `{ "type": "except", "list": [1] }` または `{ "type": "only", "list": [2] }`（キャラクターIDで指定） |

### リソース消費

| フィールド                  | 型       | 必須  | 投稿者入力 | 説明                                 |
| ---------------------- | ------- | --- | --- | ---------------------------------- |
| assist_used            | boolean | ✅   | 不要 | アシストを使用するか。`inputs` から自動導出（"Assist" の有無）。データとしては保持しフィルタに使用 |
| tag                    | boolean | ✅   | 不要 | コンボ中にキャラ交代するか。`inputs` から自動導出（"Tag" の有無）。Sidekickでは必ず `false`。データとしては保持しフィルタに使用 |
| ult_gauge_point        | number  | ✅   | 不要 | ポイントキャラのアルティメットゲージ消費量。未使用なら `0`。`inputs` から自動算出を試み、不整合があれば計算式を修正 |
| ult_gauge_assist       | number  | ✅   | 不要 | アシストキャラのアルティメットゲージ消費量。未使用なら `0`。同上 |

### コンボ内容

| フィールド  | 型        | 必須  | 説明                                                        |
| ------ | -------- | --- | --------------------------------------------------------- |
| inputs | string   | ✅   | 入力表記（アシスト技の種別もここに含む）                                      |
| damage | number   | ✅   | ダメージ数値                                                    |
| tags   | string[] |     | 追加要素タグ。選択肢: `oki`（起き攻め可） / `corner_carry`（端運び） / `side_switch`（入替え） / `fury`（フューリー使用） / `hkd`（ハードノックダウン） / `limit_strike`（リミットストライク） / `crossup`（めくり） / `punish`（確反用） |

### コンボ評価

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| difficulty | 1-5 | ✅ | 難易度。★1（パルス自動）〜★2（パルス拡張）〜★3（手動低難度）〜★4（実戦コンボ）〜★5（高難度）。投稿者が入力 |
| practicality | number | — | 実用度スコア。位置・始動・ゲージ消費・タグ数から自動算出（投稿者入力なし）。Phase 2で本格実装 |
| recommended | boolean | — | おすすめコンボフラグ。管理者が手動設定。3層構造の「これだけコンボ」のキュレーションに使用。Phase 2でpracticality算出が安定したら依存度を下げる |

### メディア・投稿者

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| video_url | string | ✅ | 動画URL（YouTube時間指定リンク または Twitter/X） |
| submitter_name | string | ✅ | 投稿者名 |
| creator_name | string | | 考案者名（投稿者と異なる場合） |
| notes | string | | 補足メモ（サイド依存情報等もここに記載） |


## 投稿者の入力項目と自動導出

投稿者の負担を最小化するため、以下のフィールドは投稿時に入力を求めない。
管理者側（またはビルド時）に `inputs` 文字列から自動導出する。

| フィールド | 導出元 | 導出方法 |
|---|---|---|
| `assist_used` | `inputs` | "Assist" の文字列検索 |
| `tag` | `inputs` | "Tag" の文字列検索。Sidekick Fuseでは強制 `false` |
| `ult_gauge_point` | `inputs` | "Super" 等のゲージ消費技をカウント。Tag前後でどちらのキャラが消費したかを判定 |
| `ult_gauge_assist` | `inputs` | 同上 |
| `practicality` | 複合 | 位置・始動・ゲージ消費・タグ数から算出 |
| `pulse` | `difficulty` | `difficulty <= 2` なら `true` |

ゲージ自動算出は誤判定リスクがあるため、投稿後に管理者が確認・修正する運用。計算式にバグがあれば都度修正。

## 2XKOのコンボ特有の要素

データ設計で考慮すべき2XKO独自の要素:
- **Fuse**: 5種（Double Down / 2X Assist / Freestyle / Juggernaut / Sidekick）でコンボが根本的に変わる
- **アシストアクション**: オフスクリーンからのアシスト攻撃をコンボに組み込む
- **ハンドシェイクタグ**: コンボ中のキャラ交代
- **ダイナミックセーブ**: 防御的タグ（コンボブレイカー）
- **パートナーキャラ**: タッグ相手によってコンボルートが変わる場合がある

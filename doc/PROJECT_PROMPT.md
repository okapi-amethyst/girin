## プロジェクト概要

2XKO（Riot Gamesの2v2タッグ格闘ゲーム、F2P）のいちプレイヤーによるコミュニティ貢献プロジェクト。
**「コンボ」を軸にした参入障壁ゼロ × テクニカルさの両立コンテンツ**で新規プレイヤーを増やす。

## なぜコンボ軸か

- 対戦で負ける人が出ない（大会形式の問題を回避）
- 1人でトレモで練習できる（相手不要）
- 2XKOのFuse×タッグシステムにより、コンボの多様性が他の格ゲーに比べて圧倒的

## コンセプト：「図鑑」ではなく「自分で選ぶ技の集積」

既存Wiki型は情報羅列で「全部やらなきゃ」の強迫観念を生む。GIRINは逆:
- 難易度・状況でフィルタして自分に合ったコンボを見つける
- 気に入ったコンボを手元に置いて練習する（自分で探して集める）
- 1つ覚えたら対戦行ける前提。対戦に送り出すのがゴール
- **核心: カジュアル層に必要なのは「これだけでいいという安心感」**

## サービス構成

### Webサイト（メニュー表）
- 閲覧メイン。**ログイン不要**で全機能が使える
- コンボ閲覧、フィルタ検索、Discord共有ボタン、SEO/OGP、QRコード
- コンボ棚卸し（LocalStorage。サーバーにデータを送らない）
- コンボデータの管理は管理者1人が行う

### Discordサーバー（本店）
- コミュニティハブ。Webからの誘導先
- コンボ修正・削除依頼の受付窓口
- コンボ投稿・相談、対戦募集、クリップ共有
- 小規模スタート、管理者1人で運営

### Discord Bot（配布型・出張所）
- どのサーバーにも招待できる
- `/combo [ID]` でコンボカードをEmbed表示、`/mycombo` でWeb棚卸しページへ誘導
- 友人への共有が大事。Web→コマンドコピー→Discordに貼る流れ

### 共有の導線

```
Webでコンボを見る
  → Discordコマンドをコピー or リンクをコピー
  → 友達サーバーに貼る
  → 見た人がWebで詳細確認 → Botを招待 / Discordサーバーに参加
```

## 投稿者への報酬

コンボ動画は投稿者のYouTube時間指定リンクで掲載（再生数が還元される）。
採用コンボに「○○式」の命名権。Discordロール付与。
**投稿者がGIRINを宣伝すると自分のYouTubeも伸びるWin-Win構造。**

## 運営方針

- 管理者1人で回せる規模を維持。回らなくなったら拡張を検討
- 投稿者の連絡先は任意記入。匿名投稿も可
- コンボ修正・削除依頼はDiscordサーバーで受付（Webにフォームは置かない）

## Riot Games 規約

- ✅ 許可: パッシブ広告、寄付、無料枠ありの変革的プレミアム機能
- ❌ 禁止: ペイウォール、クラファン、Riot IPグッズ、NFT
- コンボ情報自体は必ず無料

## フェーズ

| Phase | 内容 |
|---|---|
| 0 | Notionでコンセプト検証（スキップ → Web開発に直接着手） |
| 1 | 静的Webサイト（Astro + GitHub Pages）+ Discordサーバー + Bot（JSON参照） |

Phase 2移行の判断基準: 1人管理が回らなくなったとき。

## Webサイト開発

### 役割

「メニュー表」。閲覧メインのサイト。ログイン不要で全機能が使える。

### 核心原則

- **ログイン機能なし**（Phase 1）。閲覧の心理的ハードルをゼロにする
- コンボデータの管理は管理者（1人）が行う
- Phase 1は静的サイト（Astro + GitHub Pages）
- Phase 2移行の判断: 1人管理が回らなくなったとき

### 導線設計

```
外の世界（検索 / SNS / QRコード）
  → Webサイト（コンボ閲覧）
  → 分岐:
    A. 対戦に行く（これでOK。目的達成）
    B. 友人に共有 → コマンド or リンクをコピー → Discordに貼る
    C. コンボ修正・削除依頼 → Discordへ
    D. もっと関わりたい → Discordへ
```

### 主要機能（Phase 1）

| 機能 | 備考 |
|---|---|
| コンボ閲覧 | ログイン不要 |
| フィルタ検索 | 難易度・位置・始動状況・タグ等 |
| Discord共有ボタン | コマンドコピー + リンクコピーの2種 |
| SEO / OGP | キャラ名+コンボで検索流入。OGPでDiscord/SNSプレビュー |
| QRコード | キャラ別ページへの直行。オフ会配布用 |
| My Combos | Phase 1は Coming Soon。Phase 2でLocalStorage実装 |

## Discord Bot開発

### 役割

「出張所」モデル。どのサーバーにも招待できる配布型Bot（Phase 2〜）。
コンボを友人に共有すること、自分の持ちコンボを管理することがキモ。

### 核心原則

- 1つのBotプログラムが全サーバーに存在。DiscordユーザーIDで統一管理
- Phase 1は本サーバー専用 + JSON参照。Phase 2でDB連携 + 配布型化
- 大会・募集・ロール管理などの本サーバー運営機能は別Botとして将来分離

### コマンド設計（概要）

| コマンド | 動作 | Phase |
|---|---|---|
| `/combo [コンボID]` | コンボカードをEmbed表示 | 1 |
| `/mycombo` | WebのマイコンボページURLを返す | 1 |
| `/submit` | コンボ投稿フォーム | 2 |
| `/clear` | クリア報告。共通DBに記録 | 2 |
| `/progress` | 自分のクリア数・達成率を表示 | 2 |
| `/ranking` | サーバー内ランキング | 2 |

### 共有の導線

```
Webでコンボを見る → Discordコマンドをコピー → 友達サーバーに貼る
  → 見た人がWebで詳細確認 → Botを招待 → 自分もコンボ管理
```

### Bot開発フェーズ

| Phase | Bot機能 |
|---|---|
| 1 | 本サーバー専用。/combo（Embed表示）、/mycombo（URL返却）。JSON参照 |
| 2 | DB連携。配布型に拡張。/submit、/clear、/progress、/ranking |
| 3 | 対戦会サーバー連携。サーバー間ランキング。コンテスト機能 |

## ペルソナ・UX判断

### GIRINが埋める領域

ゲーム内ランクマッチは「競争・挑戦・興奮・有能感」を満たしている。
GIRINが補完するのは以下の構造的に未充足な動機:

| 動機 | 説明 |
|---|---|
| つながり・居場所 | オンライン格ゲーは孤独。対戦→終了→切断の繰り返し |
| 発見・探求 | コンボ開発の楽しさに場がない |
| 達成・コンプリート | 進捗可視化の仕組みがない |
| 創造・表現 | 共有する場がない |
| 意味・貢献 | 誰かの役に立つ実感 |
| 自律性 | ランクマ以外の遊び方が認められていない |

### 4つのペルソナ（判断基準）

| ペルソナ | 核心ニーズ | 設計上の注意 |
|---|---|---|
| **P1: 外から来る人** | 「これだけでいいという安心感」 | 情報を絞る。低難易度1つで対戦に送り出す |
| **P2: コンプリート型** | 自分の進捗の可視化 | 順位ではなく積み上げ。対戦への自然な導線を添える |
| **P3: 課題解決型** | 困りごとの逆引き解決 | 「全部覚えなきゃ」を防ぐ。1つ解決→対戦→ループ |
| **P4: クリエイター型** | 投稿→採用→YouTube還元のWin-Win | 投稿者が宣伝すると自分も得する構造 |

### ペルソナ別導線

```
P1 → SNSでコンボ動画 → インストール → 低難易度コンボ → 対戦へ → P2/P3に移行
P2 → 難易度帯ごとにコンボを埋める → 相方募集で対戦に向かう → P3に移行
P3 → 棚卸しで弱点発見 → 難易度/状況でフィルタ → 練習 → 対戦に戻る（ループ）
P4 → コンボ投稿 → 採用 → YouTube還元 → 自発的に宣伝（好循環）
```

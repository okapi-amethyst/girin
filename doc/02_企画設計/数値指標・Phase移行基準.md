# 数値指標・Phase移行基準

> プロジェクトの状況判断と機能追加のタイミングを数字で判断するための指標

## Phase 1 → Phase 2 移行基準

**判断者:** 管理者本人（自己判断）  
**基準:** 見た目・機能が完成したと思ったとき

数値による閾値は設けない。以下の確認観点を参考にして判断する:

| 確認観点 | 内容 |
|---|---|
| UIの完成度 | フィルタ・ソート・ページング・動画パネルが意図通り動作するか |
| コンボデータ | 公開に足る最低限のコンボ数が揃っているか（ペア × Fuse 単位での判断） |
| OGP・SEO | SNS / Discord でのリンクプレビューが正しく表示されるか |
| Bot | `/combo [ID]` が本サーバーで動作するか |
| スマホ対応 | モバイルで主要操作（フィルタ・動画再生・コマンドコピー）が快適にできるか |

---

## Phase 2 で取得できるようになる指標

| 指標 | 判断に使う場面 |
|---|---|
| ページ閲覧数・流入元 | どのキャラが人気か、SEOが効いているか |
| キャラ別ページの閲覧比率 | コンボ追加の優先順位 |
| Discordサーバー人数 | コミュニティの成長 |
| mycombo登録数（コンボ別） | 定番タグの判断 |
| Bot招待サーバー数 | 配布の広がり |
| `/combo` コマンド実行回数 | Botの利用頻度 |
| コンボ投稿数 / 月 | 投稿者の活性度 |
| `#コンボ修正依頼` の投稿数 | 管理負荷の増加サイン |

---

## Phase 2 → Phase 3 移行・拡張の判断基準

| 状況 | 目安 | アクション |
|---|---|---|
| 修正依頼が週5件超 | 対応に毎日時間を取られる | モデレーター追加 or 投稿者に編集権限 |
| Discordが100人超 | 荒らし・質問対応が増える | モデレーター追加 |
| Bot招待サーバーが10超 | 個別対応が難しくなる | Bot配布型の安定化・ドキュメント整備 |

※ 目安は仮置き。運営しながら実態に合わせて調整する。

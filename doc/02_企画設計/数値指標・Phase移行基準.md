# 数値指標・Phase移行基準

> プロジェクトの状況判断と機能追加のタイミングを数字で判断するための指標

## Phase 1で取得する指標

| 指標 | 取得方法 | 判断に使う場面 |
|---|---|---|
| ページ閲覧数・流入元 | Google Analytics / Cloudflare Analytics | どのキャラが人気か、SEOが効いてるか |
| キャラ別ページの閲覧比率 | 同上 | コンボ追加の優先順位 |
| Discordサーバー人数 | Discord標準 | コミュニティの成長 |
| `#コンボ修正依頼` の投稿数 | 目視カウント | 管理負荷の増加サイン |
| GitHub Pagesのデプロイ頻度 | GitHub Actions履歴 | 自分の更新ペース |
| コンボ総数・キャラ別数 | JSONファイルから集計 | コンテンツの充実度 |

## Phase 2で取得できるようになる指標

| 指標 | 判断に使う場面 |
|---|---|
| mycombo登録数（コンボ別） | 定番タグの判断 |
| Bot招待サーバー数 | 配布の広がり |
| `/combo` コマンド実行回数 | Botの利用頻度 |
| コンボ投稿数 / 月 | 投稿者の活性度 |

## Phase移行・拡張の判断基準

ふわっとした「回らなくなったら」を数字で線引きする。

| 状況 | 目安 | アクション |
|---|---|---|
| 修正依頼が週5件超 | 対応に毎日時間を取られる | モデレーター追加 or 投稿者に編集権限 |
| コンボ総数が200超 | JSON手動管理の限界 | DB化（Phase 2）検討 |
| Discordが100人超 | 荒らし・質問対応が増える | モデレーター追加 |
| Bot招待サーバーが10超 | 個別対応が難しくなる | Bot配布型の安定化・ドキュメント整備 |

※ 目安は仮置き。運営しながら実態に合わせて調整する。

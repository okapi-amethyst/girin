# サイト構造設計

> 最終更新: 2026-03-31
> キャラ数前提: 13キャラ（12キャラ + 近日追加1名）
> 14キャラ目は実装時期未定。CHARACTER_MASTER に追加すればページは自動生成される設計とする。

## URL構造

```
/                                       トップ（キャラ選択グリッド）
/[char]/                                キャラ単体（相方選択グリッド）
/[char]/juggernaut/                     JUGGERNAUTコンボ（ソロFuse。キャラ単体ページ配下）
/[char1]/[char2]/                       ペアトップ（Fuse選択 + α：要検討）
/[char1]/[char2]/double-down/           Double Downコンボ
/[char1]/[char2]/2x-assist/             2X Assistコンボ
/[char1]/[char2]/freestyle/             Freestyleコンボ
/[char1]/[char2]/sidekick/              Sidekick（先頭キャラ始動）
/[char1]/[char2]/sidekick-alt/          Sidekick（後方キャラ始動）
/mycombo/                               My Combos（Phase 1: サンプルコンボ表示。ナビ導線なし）
/recent/                                新着コンボ（Phase 1: サンプルコンボ表示。ナビ導線なし / Phase 2: DB連携）
/submit/                                コンボ投稿（Phase 2: フォーム実装。ナビ導線なし）
```

## ペア順序ルール

- CHARACTER_MASTER の `id` 番号順（小さい方が先）
- 既存12キャラはアルファベット順、新キャラは後番追加
- 例: Ahri(1) + Yasuo(12) → `/ahri/yasuo/`、逆順 `/yasuo/ahri/` は `/ahri/yasuo/` にリダイレクト

## Sidekick の方向性問題と解決

Sidekick Fuse はポイントキャラがアシストを呼ぶ一方通行の関係。ペアページは `id` 順で統一するため、始動キャラクターが異なる2パターンが発生する。

- `/ahri/yasuo/sidekick/` → Ahri始動（AhriがポイントでYasuoをアシストとして呼ぶ）
- `/ahri/yasuo/sidekick-alt/` → Yasuo始動（YasuoがポイントでAhriをアシストとして呼ぶ）

他のFuse（Double Down, 2X Assist, Freestyle）ではHandshake TagやTag Launcherでキャラ交代が可能なため、始動キャラの区別はコンボデータ側の `point_character` で管理し、URL分離は不要。

- **要検討:** Fuse選択だけでは面白くない。以下の候補を含め内容を検討する

  - ペアの特徴サマリー
  - その他

- Fuse情報ヘッダー

- ソロFuse用の簡易フィルターバー（アシスト/交代/DUOフィルタなし）

### My Combos `/mycombo/`
- Phase 1: サンプルコンボ表示（ナビ導線なし）

### 新着コンボ `/recent/`
- Phase 1: サンプルコンボ表示（ナビ導線なし）

## フィルター仕様

ペアページ・Fuse別ページで使用。ペア + Fuseまでは URL で絞られた状態で表示。
UIパターン: **Collapsible Filter Bar**（常時表示チップ + 展開パネル）

### フィルタ状態の管理

- **URLクエリパラメータに反映する**（例: `?difficulty=3&position=corner`）
- リンク共有で同じフィルタ状態が再現される
- **Fuse タブ切替時もフィルタパラメータを引き継ぐ**（難易度はユーザーレベルに直結するためリセットしない）

### 常時表示（バー部分）

ページを開いた時点でチップとして見えている。タップで即フィルタ。

| # | フィルター | 対応フィールド | UI種別 | 常時表示の理由 |
|---|---|---|---|---|
| 1 | 難易度 | `difficulty` | チップ選択（★1〜★5） | P1〜P3全員が最初に触るフィルタ |
| 2 | 始動状況 | `starter` | チップ / ドロップダウン | コンボ選びの基本軸。始動を変えると別コンボ |
| 3 | 位置 | `position` | チップ選択 | 中央/端で全く別のコンボになる |
| — | ソート | （下記参照） | ドロップダウン | 並び順の切替は常時アクセスしたい |

### 展開パネル（「その他フィルタ」ボタンで開閉）

普段は隠れている。使い込むプレイヤー（P2〜P3）向け。

| # | フィルター | 対応フィールド | UI種別 |
|---|---|---|---|
| 4 | アシスト使用 | `assist_used` | トグル |
| 5 | 交代（タグ）使用 | `tag` | トグル |
| 6 | ~~パルス使用~~ | — | 難易度★1-2にPulseバッジ表示で代替。フィルタUIからは除外 |
| 7 | デュオ限定 | `duo_only` | トグル |
| 8 | ポイントゲージ消費 | `ult_gauge_point` | 範囲（0〜） |
| 9 | アシストゲージ消費 | `ult_gauge_assist` | 範囲（0〜） |
| 10 | 追加要素タグ | `tags` | チップ複数選択 |

| 値 | 内容 |
|---|---|
| 対地上 | 地上ヒット始動 |
| 対空 | 2H対空始動 |

| 値 | 内容 |

※ ブレイク対策・ブレイクガード後などの特殊状況はフィルタではなくコンボのノート欄で対応。将来のサイト内検索で対応する。

## コンボの「おすすめ」キュレーション

### 背景
プロジェクト概要の3層構造（第1層「これだけコンボ」）を実現するために、大量のコンボの中から「まずこれ」を示す仕組みが必要。practicality ソートで自然に上位に来るのが理想だが、算出ロジックは未検証（Phase 2）。

### 方針
- コンボデータに `recommended: true` フラグを持たせ、手動キュレーションで対応
- ペアトップページの要検討要素としておすすめコンボ表示を候補に入れる
- Phase 2 で practicality 算出が安定したらフラグに頼らない運用に移行

## 動画パネル（Sticky Media Companion）

コンボカードの ▶ ボタンをクリックすると、コンボ一覧の上に動画パネルが sticky 表示される。

### 構造
- YouTube iframe 埋め込み（常に1つのみ。カード切替で差し替え）
- 全入力表記（短縮なし）
- コンボ情報ラベル（#ID + 難易度 + ダメージ）
- × 閉じるボタン

### レイアウト
- **モバイル（< 840px）**: 縦並び。動画パネルは sticky で上部固定、その下にフィルター、さらに下にカード一覧
- **PC（≥ 840px）**: 2カラム。左にカード一覧（480px固定）、右にサイドバー（動画パネル+フィルター）。サイドバーは sticky。動画が開くとフィルターは動画の下にずれる

### 挙動
- 動画未表示時 → カード本体クリックで動画パネルを開く
- 動画表示中 → カード外・別カード本体クリックで閉じる
- 動画表示中 → 別カードの▶ボタンでそのカードの動画に切替
- 同じカードの▶ボタン再クリック → パネルを閉じる
- ×ボタン → パネルを閉じる、iframe src を空にして再生停止
- **Escキー → パネルを閉じる**
- イベント委譲（document レベル）で実装し、DOMContentLoaded 内で初期化

### 将来検討
- ループ再生（YouTube start/end パラメータ or 自前mp4）
- 入力ステップハイライト（動画の再生位置に合わせて入力表記を1手ずつ強調）
- 自前動画ホスティング（mp4直接配信）による再生コントロール自由度の向上

## 難易度とペルソナの対応

サイトUIでは難易度★1〜5のみ使用。「第1〜3層」はプロジェクト概要のコンテンツ設計思想であり、データ属性としては持たない。

| 難易度 | 内容                       | 主な対象ペルソナ     |
| --- | ------------------------ | ------------ |
| ★1  | パルスコンボ（ボタン連打の自動コンボ）      | P1（入口）       |
| ★2  | パルスの同一ボタン連打から脱却して伸ばしたコンボ | P1（ステップアップ）  |
| ★3  | パルスではないが低難易度の手動コンボ       | P1→P2/P3への移行 |
| ★4  | 実戦コンボ（状況対応の手札）           | P3（課題解決）     |
| ★5  | 高難度・ダメージ追求・自己表現          | P4（やりこみ）     |

※ 状況別（端、対空、崩し後など）の分類はタグで行う。難易度とは別軸。

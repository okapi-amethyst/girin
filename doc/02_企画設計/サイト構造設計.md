# サイト構造設計

> 最終更新: 2026-04-11
> キャラ数前提: 13キャラ
> 14キャラ目は実装時期未定。CHARACTER_MASTER に追加すればページは自動生成される設計とする。

## URL構造

```
/                                       トップ（キャラ選択グリッド）
/[char]/                                キャラ単体（相方選択グリッド）
/[char]/juggernaut/                     JUGGERNAUTコンボ（ソロFuse。キャラ単体ページ配下）
/[char1]/[char2]/                       ペアトップ（Fuseタブ選択 + 最新コンボ5件。フィルターなし）
/[char1]/[char2]/double-down/           Double Downコンボ
/[char1]/[char2]/2x-assist/             2X Assistコンボ
/[char1]/[char2]/freestyle/             Freestyleコンボ
/[char1]/[char2]/sidekick/              Sidekick（始動方向は point_character フィルターで区別）
/combo/[id]/                            コンボ単体ページ（練習モード・共有着地点）
/mycombo/                               My Combos（Phase 1: サンプルコンボ表示。ナビ導線なし）
/recent/                                新着コンボ（Phase 1: サンプルコンボ表示。ナビ導線なし / Phase 2: DB連携）
/submit/                                コンボ投稿（Phase 2: Webフォーム実装。唯一の投稿窓口。ナビ導線なし）
```

## ペア順序ルール

- CHARACTER_MASTER の `id` 番号順（小さい方が先）
- 既存12キャラはアルファベット順、新キャラは後番追加
- 例: Ahri(1) + Yasuo(12) → `/ahri/yasuo/`、逆順 `/yasuo/ahri/` は生成しない（404）
- 内部ナビゲーションと Bot は常に正順 URL を生成するため、逆順 URL がユーザーに届くことは想定しない

## Sidekick の始動方向について

Sidekick Fuse はポイントキャラがアシストを呼ぶ一方通行の関係。どちらのキャラが始動するかでコンボが全く異なる。

- 旧設計: `/sidekick/`（char1始動）と `/sidekick-alt/`（char2始動）の2URL
- **現設計: `/sidekick/` に統合。始動方向は `point_character` フィールドで管理し、ページ上のポイントキャラフィルターで絞り込む**

他のFuse（Double Down, 2X Assist, Freestyle）でも `point_character` は存在するため、ポイントキャラフィルターは全ペアFuseページで共通して提供する。Sidekick では特に重要なフィルターになる。

- 1ページ1コンボなので、Fuse別ページの動画パネルのような一時表示ではなく、動画がメインコンテンツ
- Discord Bot `/combo [ID]` の埋め込みリンク先、SNS 共有用 OGP の着地点としても機能

- Phase 1: サンプルコンボ表示（ナビ導線なし）

  - Phase 1: Discord への誘導
  - Phase 2以降: サイト内投稿フォーム、またはページ内でフォーム展開

## 動画パネル（Sticky Media Companion）

コンボカードの ▶ ボタンをクリックすると、コンボ一覧の上に動画パネルが sticky 表示される。

**役割の位置づけ:** 動画パネルは **「探すモード」** のための補助表示。「動画を見ながら別のコンボも一覧で選べる」ことを目的とし、フィルタ・一覧が主役のページで動画を軽くチェックする用途。腰を据えて1コンボを練習する体験は `/combo/[id]/`（コンボ単体ページ）に委譲する。

### 構造
- YouTube iframe 埋め込み（常に1つのみ。カード切替で差し替え）
- 入力表記（全表記・横書き・折り返しあり）
- コンボ情報ラベル（#ID + 難易度 + ダメージ）
- × 閉じるボタン
- 「このコンボを練習する」リンク（コンボ単体ページへ）

### レイアウト
- **モバイル（< 840px）**: 縦並び。動画パネルは sticky で上部固定、その下にフィルター、さらに下にカード一覧
- **PC（≥ 840px）**: 2カラム。左にカード一覧（480px固定）、右にサイドバー（動画パネル+フィルター）。サイドバーは sticky。動画が開くとフィルターは動画の下にずれる

詳細レイアウト・入力表記の表示形式は `03_詳細設計/Fuse別ページ.md` を参照。
練習モード向けの大画面動画・番号付き縦リストの入力表記は `03_詳細設計/コンボ単体ページ.md` を参照。

### 挙動
- 動画未表示時 → カード本体クリックで動画パネルを開く
- 動画表示中 → カード外・別カード本体クリックで閉じる
- 動画表示中 → 別カードの▶ボタンでそのカードの動画に切替
- 同じカードの▶ボタン再クリック → パネルを閉じる
- ×ボタン → パネルを閉じる、iframe src を空にして再生停止
- **Escキー → パネルを閉じる**
- イベント委譲（document レベル）で実装し、DOMContentLoaded 内で初期化

### 将来検討
- ループ再生（YouTube start/end パラメータ or 自前mp4）
- 入力ステップハイライト（動画の再生位置に合わせて入力表記を1手ずつ強調）
- 自前動画ホスティング（mp4直接配信）による再生コントロール自由度の向上

## 広告配置方針

> Phase 2〜で AdSense 等を導入する想定。Phase 1 は一部実装済みプレースホルダーあり。
> **新規ページを設計するときは、このセクションを確認してから個別ページの広告セクションを書くこと。**

### Riot Games 規約上の制約

- ✅ パッシブ広告（表示型バナー）: 許可
- ❌ インタースティシャル・ポップオーバー・ゲートウェイ広告: 禁止（コンテンツへのアクセスをブロックする形式）
- コンボ情報自体は必ず無料で閲覧できること

### サイドバー広告（Base.astro レベル・全ページ共通）

viewport **1360px 以上**で、コンテンツ（960px）の左右外側に固定サイドバー広告を配置（`Base.astro` で管理）。

```
[左広告 160px] ← 36px → [コンテンツ 960px] ← 36px → [右広告 160px]
                   合計 min-width: 1360px で表示
```

- 実装済み（Phase 1 はプレースホルダー表示、Phase 2 で AdSense タグに差し替え）
- **1360px 以上ではコンテンツ内広告を `display: none` にする**（サイドバーで収益を取るため重複しない）
- z-index: 10（headroom-nav: 200 より低い）

### コンテンツ内広告の共通ルール

#### 全ページ共通の禁止エリア

| 禁止エリア | 理由 |
|---|---|
| ページ最上部（パンくず・ヘッダーの上） | 表示直後に広告が来ると離脱率が上がる |
| `position: fixed` 要素（headroom-nav 等）の上下 | スクロール時に重なって操作不能になる |
| `position: sticky` 要素（video-panel 等）の近傍 | 動画・フィルター操作を直接妨害する |
| コンボカード・キャラカードの途中への混入 | コンテンツと誤認させる |
| フィルターバー・インタラクティブ操作エリアの内部 | 操作の邪魔になる |

#### 安全な候補位置の原則

1. **ページ読み切り後（フッター直前）が最優先** — フィルタ・一覧・動画を全て読んだ後でのみ広告が表示される
2. **セクション間の自然な区切り** — ユーザーが「次のアクション」を選ぶ遷移点に置く
3. **操作の邪魔をしない** — ボタン・リンクと誤認させない位置・サイズを選ぶ
4. **最初は1箇所だけ導入** — 複数箇所は収益が安定してから段階的に追加する

## 難易度とペルソナの対応

サイトUIでは難易度★1〜5のみ使用。「第1〜3層」はプロジェクト概要のコンテンツ設計思想であり、データ属性としては持たない。

| 難易度 | 内容                       | 主な対象ペルソナ     |
| --- | ------------------------ | ------------ |
| ★1  | パルスコンボ（ボタン連打の自動コンボ）      | P1（入口）       |
| ★2  | パルスの同一ボタン連打から脱却して伸ばしたコンボ | P1（ステップアップ）  |
| ★3  | パルスではないが低難易度の手動コンボ       | P1→P2/P3への移行 |
| ★4  | 実戦コンボ（状況対応の手札）           | P3（課題解決）     |
| ★5  | 高難度・ダメージ追求・自己表現          | P4（やりこみ）     |

※ 状況別（端、対空、崩し後など）の分類はタグで行う。難易度とは別軸。

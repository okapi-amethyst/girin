# Discord Bot設計：出張所と本店モデル

> 作成日: 2026-03-26
> ステータス: 設計段階

---

## コンセプト

Botを1サーバー専用にせず、**どのサーバーにも招待できる配布型Bot**として設計。
外部サーバーが「出張所」、メインサーバーが「本店」の役割を持つ。

---

## 技術的な仕組み

### なぜサーバーをまたいで進捗共有できるか

```
Discord Botは「1つのプログラム」が全サーバーに存在する

  友達サーバーA   対戦会サーバーB   本サーバー
       │              │              │
       └──────┬───────┘──────────────┘
              │
        同じBot（1つ）
              │
           共通DB
    ┌─────────────────┐
    │ ユーザーID: 12345 │ ← DiscordユーザーIDは全サーバー共通
    │ ヤスオ基本: ✅    │
    │ エコー基本: ✅    │
    │ クリア数: 2       │
    └─────────────────┘
```

- DiscordのユーザーIDはグローバルに一意
- どのサーバーで `/clear` しても同一ユーザーとして共通DBに記録
- 友達サーバーでクリアしたものが本サーバーの全体フィードにも反映

## コマンド設計

### 全サーバー共通（出張所でも使える）

| コマンド | 動作 | 例 | Phase |
|---|---|---|---|
| `/combo [コンボID]` | コンボカードをEmbed表示（キャラ名・入力・ダメージ・難易度・動画リンク・Webリンク） | `/combo 42` | 1 |
| `/mycombo` | WebのマイコンボページURLを返す | `/mycombo` | 1 |
| `/clear [コンボID]` | クリア報告。共通DBに記録＋そのサーバーに通知 | `/clear 42` | 2 |
| `/progress` | 自分のクリア数・達成率を表示 | `/progress` | 2 |
| `/progress @ユーザー` | 指定ユーザーの進捗を表示 | `/progress @まさし` | 2 |
| `/ranking` | **そのサーバー内**のメンバーだけのランキング | `/ranking` | 2 |

### 本サーバー限定（本店でしかできない）

| コマンド | 動作 | 理由 |
|---|---|---|
| `/ranking global` | 全ユーザーの総合ランキング | 集中（全体を見る場） |
| `/contest submit` | コンテストにコンボを投稿 | 集中（全員が同じ場で） |
| `/contest vote` | コンテストの投票 | 集中 |
| `/duo [キャラ] [ランク帯]` | デュオ相手の募集 | 集中（人数が必要） |
| `/event` | 対戦会の告知・一覧 | 集中（広く告知） |

### コマンドの動きの具体例

```
■ 友達サーバーAで /clear 42 を実行

友達サーバーAに表示：
  🎉 ○○ がヤスオ基本コンボ (#42) をクリア！（通算5個目）
  → 友達が「おめ！」とリアクション

本サーバーの #全体クリアフィード に自動投稿：
  ○○さんがヤスオ基本コンボ (#42) をクリア！
  → 知らない人の達成も流れる → 「自分もやろう」の刺激

共通DBを更新：
  ○○のクリア数: 4→5
  → 5個達成でロール自動付与（本サーバーのロールが更新される）
```

```
■ 対戦会サーバーBで /ranking を実行

対戦会サーバーB内のメンバーだけで表示：
  1位 まさし  15個
  2位 あきら  11個
  3位 ゆうき   7個
  「全体ランキングはこちら → [本サーバーリンク]」

■ 本サーバーで /ranking global を実行

全サーバー横断の総合ランキング：
  1位 まさし  15個（対戦会B）
  2位 たろう  12個（友達サーバーA）
  3位 あきら  11個（対戦会B）
  ...
```

## ロール設計（本サーバー）

Botがクリア数に応じて本サーバーのロールを自動付与。

| 条件 | ロール名（案） | 色 |
|---|---|---|
| 初めてのクリア報告 | GIRINデビュー | 白 |
| 10個クリア | GIRINブロンズ | 銅 |
| 全キャラ第1層制覇 | GIRINシルバー | 銀 |
| 全キャラ第2層制覇 | GIRINゴールド | 金 |
| コンテスト採用1回 | コントリビューター | 緑 |
| コンテスト採用3回 | GIRINマスター | 紫 |
| 殿堂入り（特別貢献） | レジェンド | 虹 |

※ クリア系ロールとコンテスト系ロールは別軸。「使う人」と「作る人」両方を評価。

## 分散の効果：なぜ外部サーバーが本サーバーを強くするか

```
外部サーバーで起きること         → 本サーバーへの導線
━━━━━━━━━━━━━━━━━━━━    ━━━━━━━━━━━━━━━━━━━━
/combo で便利に使う              → 「もっとコンボないの？」
/ranking で内輪のランキング見る   → 「全体では何位？」
/clear でクリア報告              → 「ロール欲しい」
Bot経由でコンテスト告知が届く     → 「参加したい」
```

内輪の雑談（「昨日の対戦やばかった」「週末遊ぼうぜ」）は
元々本サーバーでは起きないもの。Botがあることで
**内輪の会話の中にGIRINが自然に存在する状態**を作れるのが利点。

## 対戦会サーバー連携の設計

対戦会主催者にBot導入メリットがある設計にする。

| 主催者のメリット | 仕組み |
|---|---|
| 参加者がコンボを調べやすい | /combo が使える |
| メンバーの成長が可視化される | /ranking でサーバー内順位 |
| 対戦会の宣伝になる | 本サーバーの全体フィードに対戦会名が露出 |
| 対戦会同士のゆるい競争 | サーバー間の平均クリア数比較（将来機能） |

## 開発フェーズ

> プロジェクト全体のPhase定義は PROJECT_PROMPT.md を参照。Bot機能は各Phaseに沿って段階的に追加。

| プロジェクトPhase | Bot機能 |
|---|---|
| Phase 1 | 本サーバー専用。/combo, /mycombo（JSON参照） |
| Phase 2 | DB連携。配布型に拡張。/submit, /clear, /progress, /ranking。全体フィード連携 |
| Phase 3 | 対戦会サーバー連携。サーバー間ランキング。コンテスト機能 |

# 入力表記記法リファレンス

> GIRINの「入力」フィールドで使用する入力表記の定義・ルール集。
> コンボ1枚フォーマット.md の `入力表記` 欄の参照先。
> 2XKOMBOの記法をベースに、Dustloop標準に合わせて読みやすさを調整。

---

## 1. 攻撃ボタン

2XKOは6ボタン構成。

| 記号 | 英語名 | 意味 |
|------|--------|------|
| `L`  | Light  | 軽攻撃 |
| `M`  | Medium | 中攻撃 |
| `H`  | Heavy  | 強攻撃 |
| `S1` | Special 1 | 必殺技1 |
| `S2` | Special 2 | 必殺技2 |
| `T`  | Team   | チームボタン（タッグ・アシスト操作全般）|

> **備考**: ニュートラル(5)との組合せで `5H → H`（ニュートラル強攻撃）のように5を省略するのが標準。

---

## 2. 方向入力（テンキー記法）

キーボードのテンキーをイメージして方向を数字で表す。Dustloop/GGSTと同じ標準記法。

```
7 8 9
4 5 6
1 2 3
```

| 数字 | 方向 | 矢印 |
|------|------|------|
| `1` | ↙ 後ろ下 | Down-Back |
| `2` | ↓ 下     | Down |
| `3` | ↘ 前下   | Down-Forward |
| `4` | ← 後ろ   | Back |
| `5` | 　ニュートラル | Neutral（通常省略） |
| `6` | → 前     | Forward |
| `7` | ↖ 後ろ上 | Up-Back |
| `8` | ↑ 上     | Up |
| `9` | ↗ 前上   | Up-Forward |

**よく出る複合例:**
- `2M` = しゃがみ中攻撃
- `6H` = 前強攻撃
- `66L` = 前ダッシュ弱攻撃
- `44` = 後ろダッシュ単独
- `29` = ハイジャンプ（`2`→`9`の連続入力。コンボ内では `29 > j.L` のようにコネクタで続く）

---

## 3. 修飾子（プレフィックス）

入力の前に付けて条件を指定する。複数組み合わせ可能（内側から順に読む）。

| 記号      | 意味             | 例         | 読み方     |
| ------- | -------------- | --------- | ------- |
| `j.`    | 空中（ジャンプ中）入力    | `j.H`     | 空中強攻撃   |
| `dl.`   | ディレイ（少し遅らせる）   | `dl.M`    | ディレイ中攻撃 |
| `[X]`   | ホールド（押しっぱなし）   | `[H]`     | 溜め強攻撃   |
| `{X}`   | 少しチャージ（ちょい長押し） | `2{H}`    | 微溜め下強攻撃 |
| `mash.` | 連打して最大ヒット      | `mash.S2` | S2を連打   |


**複合例:**
- `j.2H` = 空中↓強攻撃（空対地技など）
- `dl.j.S2` = ディレイ空中S2
- `6[S1]` = 前S1でS1をホールド
- `j.[H]` = 空中でHをホールド

## 4. 接続記号

コンボのステップ間をつなぐ記号。

| 記号            | 意味                        | 例                      |
| ------------- | ------------------------- | ---------------------- |
| `>`           | キャンセル・連続技でつなぐ             | `M > H > S1`           |
| `,`           | リンク（前の技のリカバリー後に接続）        | `j.S2 , L`             |
| `~`           | フォローアップ入力 / 数フレーム以内に続けて入力 | `S1~H`                 |
| `+`           | 同時押し                      | `S1+S2`                |
| `(X)`         | **オプション入力**（省略可能）         | `(L >) M > H`          |
| `duo.(X > Y)` | **残キャラ入力ブロック**（デュオ専用）     | `T > duo.(S1 > M) > H` |
| `MOVE(N)`     | **ヒット数指定**（N ヒット目でキャンセル）  | `S2(2) > S1+L`         |

> **`>` とリンク `,` の使い分け**: 実際のコンボ記事では区別せず `>` を使うケースが多いが、厳密にリンクであることを示したい場合は `,` を使う。どちらでも誤りにはならない。

### 残キャラ入力ブロック `duo.()`

デュオ専用コンボで、タッグ後にスクリーンに残ったキャラが入力する部分を `duo.()` で囲む。
`duo.()` の外はメインで操作しているキャラの入力、`duo.()` の中は残ったキャラの入力。

```
... > T > duo.(S1 > S1) > M > H > ...
```

- 括弧内と括弧外の入力は並行して行われる（時間軸は厳密に分けられない）
- ソロコンボや通常のアシスト呼び出し（`6T`）では使用しない
- 表示側でのカラーハイライト等はコンボカード設計に委ねる

### ヒット数指定 `MOVE(N)`

多段技を途中でキャンセルする場合、何ヒット目でキャンセルするかを直後の `(数字)` で指定する。

```
S2(2) > S1+L   # S2が3ヒットする技だが2ヒット目にスーパーでキャンセル
```

**`()` の使い分け:**
- `duo.(...)` → デュオ残キャラブロック
- `MOVE(数字)` → ヒット数指定（`S2(2)`）
- `(入力)` 単体 → オプション入力（省略可能）

### 択一・代替ルートについて

`/` 記号による択一記法（例: `M > 2M / 2M > M`）は採用しない。  
順不同・どちらでも可といった情報は**補足メモ欄**に記載する。

## 5. チームボタンアクション

`T`（チームボタン）は入力の組み合わせによって以下のアクションに分かれる。

| 記号              | アクション名           | 条件                                   | 意味                          |
| --------------- | ---------------- | ------------------------------------ | --------------------------- |
| `T`             | ハンドシェイクタッグ       | チームメイトが画面上にいる時                       | チームメイトとタッグ交代                |
| `[T]`           | クイックタッグ          | チームメイトが画面外の時                         | チームメイトを呼び出してタッグ             |
| `T` / `6T`      | アシストアクション（前）     | チームメイトが画面外の時、6Tは2X Assistの2回目のアシストの時 | パートナーの前アシスト攻撃               |
| `4T`            | アシストアクション（後ろ）    | —                                    | パートナーの後ろアシスト攻撃              |
| `2T`            | タッグランチャー         | アシストがKO時以外の時                         | タッグランチャー発動                  |
| `2T`            | スーパーアシスト         | アシストがKO時のみ                           | パートナーがKOされている時に発動するスーパーアシスト |
| `2T`            | イジェクト            | Juggernautフューズ使用時                    | Juggernautフューズの1ゲージ使用する特殊攻撃 |
| `T+S1` / `T+S2` | ブレイク / フューリーブレイク | ブレイクメーター満タン時                         | ブレイク発動                      |

> **チャージドアシスト**（`T`長押し または `4T`、Sidekick Fuse限定）: 現状GIRINコンボ表記での記法未定。Sidekickページの設計時に決定する。

## 6. 2XKOMBO記法との対応

2XKOMBOのNUMPADモードのテキスト記法とGIRINの記法は概ね近いが、以下の差分がある。GIRINの投稿フォームはペースト時に自動変換する（詳細は `コンボ投稿ページ_入力UI.md` のペースト互換変換セクション参照）。

| 2XKOMBO記法     | GIRIN記法     | 備考            |
| ------------- | ----------- | ------------- |
| `D`           | `66`        | 前ダッシュ         |
| `BD`          | `44`        | 後ろダッシュ        |
| `r.`          | `66`        | ダッシュ攻撃        |
| `MOVE(mash)`  | `mash.MOVE` | 連打            |
| `w.`          | （変換なし）      | GIRIN に対応記法なし |
| ` \| `        | （削除）        | 改行はしない        |
| `T:CHARACTER` | `T`         | キャラクター表記はとる   |

**例: 実際のコンボ（Yasuo+Jinx）をGIRIN記法に変換**

2XKOMBO原文:
```
M > H > S1 > 2S1 > M > j.S1 > 2S1 > dl.M > j.2H > dl.j.S2 > D > M > H > j.2H > j.S2
> T > BD > BD > [6S1] > 2S1 > BD > [6S1] > 2T:JINX > j.H > j.[H] > S1+S2
```

GIRIN記法（変換後）:
```
M > H > S1 > 2S1 > M > j.S1 > 2S1 > dl.M > j.2H > dl.j.S2 > 66 > M > H > j.2H > j.S2
> T > 44 > 44 > [6S1] > 2S1 > 44 > [6S1] > 2T > j.H > j.[H] > S1+S2
```

## 7. GIRINサイトでの表記方針

### 基本ルール
1. **テンキー記法を使用**。矢印記号（↓など）は使わない（テキスト処理・検索のため）
2. 誰が動いているかの視覚的な解説は動画・GIFが主な手段。デュオの詳細は補足メモ欄を活用

### デュオ専用コンボでの残キャラブロック

タッグ後に残ったキャラの入力がコンボに絡む場合は `duo.()` でブロック化する。

```
M > H > S1 > T > T > duo.(S1 > M) > M > H > S1+S2
```

### データとしての保存形式
DBには入力テキストをそのまま保存。`duo.()` のパースと表示カラーハイライトは実装側で処理する。

## 8. 注意事項・例外

- **サイド依存**: 入力方向は「自キャラ側を左」として記述（Dustloop標準）。
- 複数ヒット技のキャンセルタイミングは注記してもらう
- **キャラクター略称**: 本ドキュメントではフルネームを使用するが、サイト実装時は CHARACTER_MASTER.md のIDで管理。
- **リリース記法**: Dustloop標準では `]X[` でボタンのリリースタイミングを指定するが、2XKOでは未確認のため本記法は使用しない。
- **ジャンプキャンセル `jc`**: 正式記法としては採用しない。`>8`（上ジャンプ）や `>9`（前ジャンプ）の表記揺れとして認識できる範囲で許容する。
- **択一記法 `/`**: 採用しない。代替ルートや順不同の情報は補足メモ欄に記載する。

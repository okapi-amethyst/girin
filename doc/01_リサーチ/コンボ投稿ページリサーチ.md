## 2. 2XKOMBO フォーム分析（実機確認済）

### 2-1. 入力フィールド

| フィールド | 必須 | 種別 | 値・選択肢 |

| SELECT FUSE | — | セレクト（選択画面へ）| Fuse一覧（任意）|
| NAME | ✅ | テキスト入力 | コンボ名称 |
| DIFFICULTY | ✅ | ドロップダウン | `EASY` / `MEDIUM` / `HARD` |
| DAMAGE | — | 数値入力 | |
| METER COST | ✅ | ドロップダウン | `0 BARS` / `1 BAR` / `2 BARS` / `3 BARS` |
| METER GAIN | — | 数値入力 | ゲージ回収量（GIRINに相当フィールドなし）|

**GIRINにはないフィールド:** METER GAIN（ゲージ回収量）。始動・位置・デュオ限定は 2XKOMBO にはない。

### 2-2. ビジュアルボタン入力（実機確認）

**表示モード切替:** NOTATION / ICON / NUMPAD の3モード。ICONモードでは矢印アイコン表示、NUMPADモードではテンキー記法テキスト表示。

**NUMPAD NOTATION テキスト欄:** ページ上部に「NUMPAD NOTATION (EDITABLE) - CLICK APPLY TO CONVERT...」というテキストエリアがあり、テンキー記法を直打ちして `APPLY` ボタンでビジュアル表示に変換できる。

**ボタン構成（実機確認）:**

| カテゴリ | ボタン | 記法 / 動作 |
|---|---|---|
| 方向 | 8方向パッド（↖↑↗ ← → ↙↓↘）| `7`〜`9`, `4`, `6`, `1`〜`3`（5/Nは非表示）|
| 攻撃 | `L` `M` `H` | 各攻撃ボタン |
| タグ | `T` | タッグ交代 |
| 接続 | `~` | `~`（フォローアップ）|
| 必殺技 | `S1` `S2` | 各スペシャル |
| ダッシュ | `>>` `<<` | 前ダッシュ / 後ダッシュ |
| 修飾子 | `DELAY` | `dl.` |
| 接続演算子 | `+` | 同時押し `+` |
| キャンセル | `>` | キャンセル接続 `>` |
| 修飾子 | `HOLD` | `[X]`（ホールド）|
| 修飾子 | `AIR` | `j.`（空中入力）|
| 条件 | `OR` | 代替入力（どちらでも可）|
| 修飾子 | `HIT#` | ヒット数指定 `(N)` |
| 修飾子 | `MASH` | `mash.` |
| 修飾子 | `WHIFF` | `w.` |
| 修飾子 | `RUN` | `r.` |
| 改行 | `NEW LINE` | コンボを複数行に分割 |

### 2-3. 出力・共有

- `UNDO` / `RESET`（赤）— 直前取消 / 全消去
- `COPY NOTATION` — 記法テキストをクリップボードへ
- `SAVE IMAGE`（サイズスライダー付き、デフォルト56px）— コンボ画像書き出し
- `SHARE COMBO` — 共有リンク生成
- `SAVE COMBO` — アカウントに保存

### 2-4. アカウント・認証モデル（実機確認 2026-04-16）

**Discord ログイン必須の機能:**
- `SAVE COMBO`（自分のコンボ一覧に保存）
- My Combos 閲覧（保存済みコンボの管理）

**投稿フロー:**
- コンボを作成 → `SAVE COMBO` でアカウントに保存 → 保存済みコンボの詳細ページから `SHARE` という体裁
- 「投稿」と「My Combosへの保存」が同一アクションとして設計されている
- ゲスト（未ログイン）は作成・プレビューはできるが保存・共有はできない

**GIRIN との設計の違い:**
- 2XKOMBOは「保存 = 投稿」の1段階モデル。GIRINは「投稿 → JSON/DBに追加」と「My Combos（ブックマーク）」が独立した2段階モデル
- 2XKOMBOはログイン必須で投稿者が常に確定する。GIRINはログイン任意で未ログイン投稿も許容する方針
- GIRINでログイン任意を採用する理由: 人気がない初期段階での投稿障壁を下げる。My Combosのクラウド同期はDiscordログインを要する機能として Phase 3 で提供予定

### 2-5. 確認できなかった項目

- TAGS の選択肢詳細
- SELECT FUSE の選択肢一覧（Juggernautや No Fuse が含まれるか）

## 4. 両サイトに共通するパターン

| パターン | GIRIN への示唆 |
|---|---|
| テキスト入力＋ビジュアルボタン併用 | 採用。テキスト直打ちと補助ボタンの併用 |
| **テキスト欄に直打ちして変換するモードも併設**（2XKOMBO確認）| 採用。`NUMPAD NOTATION` 欄に貼り付けて `APPLY` → GIRINでも「テキスト貼り付けモード」として実装できる |
| キャラクターは事前選択 or 必須セレクト | 採用。ただし GIRIN は2キャラ（ポイント＋アシスト）選択 |
| 難易度・ゲージ消費の分類メタデータ | 採用。GIRIN は既に `difficulty` `ult_gauge_*` を定義済 |
| UNDO / RESET ボタン | 採用。入力ミス時の巻き戻し必須 |
| 方向と攻撃ボタンは横並びの専用パッドUI | 採用。2XKO専用UIとして6ボタン+9方向を常時表示 |
| 修飾子はトグル/プレフィックス単位のボタン | 採用。`j.` `dl.` `[X]` 等を1クリックで挿入 |
| 表示モード切替（NOTATION / ICON / NUMPAD）| 参考。GIRIN の表示切替（短縮/全表記）と方向性は同様 |
| プレビュー・記法コピー・画像書き出し | 部分採用。Phase 1 はプレビューのみ（画像書き出しは不要）|

| 考案者名         | —                 | —                | ⭕️ 任意                              |

**所見:**
- GIRIN のメタデータ設計（Fuse必須/始動/位置/デュオ限定）は既存2サイトよりリッチ
- 既存サイトから移植できるのは主に **ビジュアルボタンによる入力表記生成UI** と **テキスト直打ちモード（貼り付け→APPLY変換）**
- METER GAIN（ゲージ回収量）は 2XKOMBO にあるが GIRIN にはない。2XKO では重要な情報かを要検討
- ComboForge のパッチバージョン表示（Current バッジ）は親切な設計。GIRIN の `game_version` 運用時に参考にする
- タグの詳細比較は §6 参照
- UIの詳細設計は `03_詳細設計/コンボ投稿ページ_入力UI.md` にまとめてある

### 6-2. GIRIN への採用検討

| タグ | 採用判断 | 理由 |
|---|---|---|
| **BNB** | 採用検討 | 両サイトにある。「基本コンボ・持ちコンボ候補」として有用。`corner_carry` より使用頻度が高い概念 |
| **TOD** | 採用検討 | 「このコンボで体力ゲージが尽きる」は重要情報。ダメージ数値で代替できなくもないが、タグのほうが一目でわかる |
| optimal | 低優先 | 「最適解」の判断は主観的で管理コストが高い |
| situational | 低優先 | GIRIN の `target_characters` / `position` / `starter` の組み合わせで表現可能 |
| Corner / Midscreen / Anywhere | 不採用 | GIRIN の `position` フィールドで十分にカバー済み |
| beginner / advanced / anti-air | 不採用 | `difficulty` / `starter` フィールドと重複 |

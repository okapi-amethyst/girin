## ミッション

2XKOの新規プレイヤーを増やすためのコミュニティ貢献として、
**「コンボ」を軸にした参入障壁ゼロ × テクニカルさの両立コンテンツ**を作る。

## 背景・課題

### 2XKOの現状
- 2026年1月20日にPC/PS5/Xboxで正式ローンチ（F2P）
- ローンチ直後はPS5無料DLランキング3位と好調だったが、2月には圏外に
- 2月にRiotが開発チーム約80人をレイオフ（チームの約半数）
- コンテンツクリエイターの大会もメンツが固定化（最大8人程度）
- ゲーム自体の認知度が低く「2XKO」という名前を知らない層が多い

### なぜコンボ軸か
- 対戦で負ける人が出ない（= 従来の大会形式の問題を回避）
- 1人でトレモで練習できる（= 相手が不要）
- 見ているだけでも楽しい（= 未プレイヤーにもリーチ）
- 2XKOのFuse×タッグシステムにより、コンボの多様性が他の格ゲーに比べて圧倒的

## コンセプト：「図鑑」ではなく「自分で選ぶ技の集積」

### 既存Wiki型の問題
| 問題 | 結果 |
|---|---|
| 情報が羅列される | どれからやればいいかわからない |
| 網羅的を目指す | 「全部やらなきゃ」という強迫観念 |
| テキスト中心 | 見てもイメージが湧かない |
| 更新が止まりがち | 古い情報が残る |

### GIRINの設計思想

```
図鑑（辞書型）           GIRIN（課題解決型）
───────────────────    ──────────────────
「全部載ってます」        「あなたに合うのはコレ」
全部覚える前提           1つ覚えたら対戦行ける前提
トレモに閉じこもる        対戦に送り出すのがゴール
```

**核心：カジュアル層に必要なのは「大量の情報」ではなく「これだけでいいという安心感」**

## 3層構造

```
┌─────────────────────────────────────────┐
│  第1層：これだけコンボ（1キャラ1つ）       │ ← ここだけで対戦できる
│  「触ったらこれ」「対空はこれ」            │    カジュアル層のゴール
│  全キャラ各2〜3個だけ                     │
├─────────────────────────────────────────┤
│  第2層：状況別コンボ                       │ ← 対戦で困ったら戻ってくる
│  「端で〜」「対空から〜」「崩した後〜」     │    中級者の参照先
│  必要なものだけ見る                        │
├─────────────────────────────────────────┤
│  第3層：最適化・魅せコンボ                 │ ← やり込み勢の遊び場
│  Fuse別ルート / ダメージ比較               │    コンテスト用
│  コミュニティ投稿コンボ                    │
└─────────────────────────────────────────┘

第1層だけ見て対戦に行く人 ＝ 正しい使い方
```

## 導線設計：「状況 → 解決策 → コンボ」

キャラ別にコンボを羅列するのではなく、**対戦中の困りごとから逆引き**する。

```
例：ヤスオを使っている人向け

😩「相手に触れたけどダメージが安い」
  → これだけコンボA（動画15秒）
  → ✅ これだけ覚えたら対戦行ってOK

😩「画面端に追い詰めたのに逃げられる」
  → 画面端コンボB（動画15秒）
  → ✅ これだけ覚えたら対戦行ってOK

😩「もっと火力を伸ばしたい」（ここで初めて応用）
  → アシスト拡張コンボC
  → Fuse別の選択肢はこちら...
```

## サービス構成：「Web＝入口」「Discord＝体験の場」

### 設計原則
Webアプリで全機能を完結させない。**意図的に「ここから先はコミュニティでしかできない」体験を作る**ことで、Wikiの二の舞（見て終わり）を防ぐ。

```
Webサイト（メニュー表）           Discord（店内）
━━━━━━━━━━━━━━━━━━━━━    ━━━━━━━━━━━━━━━━━━━━━
コンボ閲覧（誰でもアカウント不要） クリア報告（人に見せる体験）
逆引き検索                       ロール・称号
SEOで検索流入                    コンテスト投稿・投票
QRコードでオフ会導線              デュオマッチング
                                 最新コンボ速報・議論
                                 対戦会告知
```

### Discord Bot「出張所と本店」モデル

Botを1サーバー専用にせず、**どのサーバーにも招待できる配布型Bot**として設計する。

```
Bot は「1つのプログラム」が全サーバーに存在
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  友達サーバーA   対戦会サーバーB   本サーバー
       │              │              │
       └──────┬───────┘──────────────┘
              │
        同じBot（1つ）
              │
           共通DB
    ┌─────────────────┐
    │ ユーザーID: 12345 │ ← DiscordユーザーIDは
    │ ヤスオ基本: ✅    │   全サーバー共通
    │ エコー基本: ✅    │
    │ クリア数: 2       │
    └─────────────────┘

→ どのサーバーで /clear しても同じユーザーとして記録
→ 友達サーバーでクリアしたものが本サーバーでも反映済み
```

### 機能の分散と集中

| 機能 | 外部サーバー（出張所） | 本サーバー（本店） | 設計意図 |
|---|:---:|:---:|---|
| コンボ検索・閲覧 | ◎ | ◎ | どこでも見れる＝利用者増 |
| クリア報告 | ◎ 内輪に通知 | ◎ 全体フィードに集約 | 内輪で「おめ！」＋全体に自動記録 |
| サーバー内ランキング | ◎ そのサーバーだけ | ◎ 全ユーザー | 内輪の競争＋全体のゆるい競争 |
| デュオマッチング | ✕ | ◎ | 人数が必要→集中 |
| コンテスト投稿・投票 | ✕ 案内リンクのみ | ◎ | 全員が同じ場で見て投票 |
| 最新コンボ速報 | △ 通知受け取り | ◎ 議論が起きる場 | 速報はどこでも、議論は本店 |
| 考案者への質問 | ✕ | ◎ | 考案者がいる場所＝本店 |
| 対戦会告知 | ✕ | ◎ | 広く告知→集中 |

### なぜ分散が本サーバーを強くするか

```
Botなし（従来）               Botあり（提案）
━━━━━━━━━━━━━━━━        ━━━━━━━━━━━━━━━━
友達サーバーで2XKOの話をする   友達サーバーにBotを入れる
→ 本サーバーの存在を知らない   → /combo で便利に使う
→ 合流しない                  → 「全体ランキング見たい」→本サーバーへ
                              → 「コンテスト参加したい」→本サーバーへ
                              → 「デュオ相手探したい」→本サーバーへ
```

外部BotはWebサイトと同じ「入口」。内輪の雑談は元々本サーバーでは起きないものなので、価値の毀損にはならない。

### 対戦会サーバーとの連携
- 対戦会主催者にもBot導入メリットがある設計
  - 参加者がコンボを調べやすい → 対戦会の質向上
  - メンバーの成長が可視化 → 運営のやりがい
  - 本サーバーに対戦会名が載る → 対戦会の宣伝になる

## 投稿者への報酬設計

「考案者クレジット」だけでは弱い。**投稿者の自己チャネルにトラフィックが還元される**構造にする。

| 報酬 | 仕組み | 満たす動機 |
|---|---|---|
| YouTube時間指定リンク | GIRINの動画枠を投稿者のYouTubeに直リンク。再生数が還元 | 創造・実利 |
| Xメンション | GIRIN公開時に「@○○さん考案」投稿。フォロワー増の導線 | 露出・つながり |
| 命名権 | 採用コンボに「○○式」の名前がつく。格ゲー文化の伝統 | 所有感・名誉 |
| Discordロール | 採用回数に応じた称号（コントリビューター→マスター→殿堂入り） | 達成・所有感 |
| 実戦採用フィードバック | 「あなたのコンボで勝てました」報告を共有 | 意味・貢献 |
| 次回お題提案権 | 月間MVPが翌月のお題を決められる | 自律性 |

**核心：投稿者がGIRINを宣伝すると自分のYouTubeも伸びるWin-Win構造**

## コンテスト連動の月次サイクル

```
① お題発表（月初）
   「今月のキャラ：○○」
   「募集：○○の"これだけコンボ"を考えてください」

② コミュニティ投稿（2週間）
   条件：初心者が3回以内に成功できること / 実戦で使えること

③ 投票・選定（1週間）
   コミュニティ投票で「今月のこれだけコンボ」を決定

④ GIRINに掲載
   選ばれたコンボが第1層に正式採用
   動画はYouTube時間指定リンクで投稿者に還元

⑤ 上級者向けは第3層へ
   高難度コンボは第3層に掲載
```

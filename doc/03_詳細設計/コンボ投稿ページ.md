## Phase 1 設計

### 概要

管理者（プロジェクトオーナー）が自身の上達用コンボをサンプルデータに追加するための投稿フォーム。Phase 1 は不特定多数への公開を前提にしない。

| 観点 | Phase 1 |
|---|---|
| 利用者 | 管理者1人のみ |
| 出力先 | JSONスニペット生成 → `combos.json` に手動追記 |
| 認証 | なし（非公開サイトのため不要）|
| ナビ掲載 | しない（URL直打ちでアクセス）|
| スパム対策 | 不要 |

### 投稿フォームフィールド（Phase 1）

| フィールド      | 必須  | 種別            | 備考                                                                        |
| ---------- | --- | ------------- | ------------------------------------------------------------------------- |
| キャラ1（ポイント） | ✅   | セレクト          | CHARACTER_MASTER から                                                       |
| キャラ2（アシスト） | —   | セレクト          | Juggernaut の場合は不要                                                         |
| Fuse       | ✅   | セレクト          | 全Fuse + No Fuse                                                           |
| 入力表記       | ✅   | テキスト＋ビジュアルボタン | 下記参照                                                                      |
| 難易度        | ✅   | ★1〜★5 選択      |                                                                           |
| 始動         | ✅   | セレクト          | ground / anti_air / pressure_mixup / tag_launcher                         |
| 位置         | ✅   | セレクト          | no_limit / center / near_corner / corner / near_corner_back / corner_back |
| デュオ限定      | ✅   | チェックボックス      |                                                                           |
| ダメージ       | —   | 数値入力          |                                                                           |
| 動画URL      | —   | テキスト入力        | YouTube URL                                                               |
| タグ         | —   | チップ複数選択       |                                                                           |
| コメント / メモ  | —   | テキストエリア       |                                                                           |
| タイトル（説明）   | —   | テキスト入力        | 15文字以内推奨                                                                  |
| 対象キャラ制限    | —   | セレクト          | all / except / only                                                       |
| 投稿者名       | ✅   | テキスト入力        | Phase 2: ログイン済みはDiscord名を自動入力（編集不可）、未ログインは手動入力                            |

**Phase 1 で省略するフィールド（データ基盤の都合）:**

| フィールド   | 省略理由                      |
| ------- | ------------------------- |
| CAPTCHA | 認証不要 |
| 審査フロー   | 管理者が直接JSON追記するため不要 |

**自動導出フィールド（フォームに表示しない）:**

| フィールド | 導出元 |
|---|---|
| `assist_used` | `inputs` に `T:` が含まれるか |
| `tag` | `inputs` に `T`（キャラ交代）が含まれるか |
| `ult_gauge_point` | `inputs` の `S1+S2` 等から推定（手入力も可）|
| `ult_gauge_assist` | 同上 |
| `duo_only` | `inputs` に `()` ブロックがある場合は自動チェックを提案 |
| `pulse` | `difficulty ≤ 2` なら自動 `true` |

### 出力方式（Phase 1）

「JSONを生成」ボタン押下で画面に JSON スニペットを表示 → 「コピー」ボタンでクリップボードへ → `combos.json` に手動追記。

```json
{
  "id": 123,
  "point_character": 5,
  "assist_character": 2,
  "fuse": "Double Down",
  "title": "端M連打パルス起点",
  "inputs": "M > H > S1 > 2S1 > j.M > j.H > j.S2",
  "difficulty": 3,
  "starter": "ground",
  "position": "center",
  "duo_only": false,
  "assist_used": false,
  "tag": false,
  "pulse": false,
  "ult_gauge_point": 0,
  "ult_gauge_assist": 0,
  "damage": 3200,
  "target_characters": "all",
  "tags": ["oki"],
  "video_url": "https://youtu.be/xxx?t=12",
  "notes": "",
  "submitter_name": "GIRINテスト"
}
```

- `id` は既存コンボ件数 + 1 を自動採番
- `assist_used` / `tag` / `ult_gauge_*` / `pulse` は `inputs` と `difficulty` から自動導出して初期値をセット（手動で変更も可）

### 段階実装（Phase 1 → Phase 2）

Phase 1 と Phase 2 の差分は **データの取扱い** のみ。UIは Phase 1 で完成形を実装する。

| 機能    | Phase 1         | Phase 2                                   |
| ----- | --------------- | ----------------------------------------- |
| 投稿先   | JSONスニペット → コピペ | DB書き込み（Supabase等）                         |
| 認証    | なし              | Discord ログイン（任意）                          |
| スパム対策 | なし              | CAPTCHA 等（人気が出た段階で検討）                     |
| 投稿者名  | 手動入力（必須）        | ログイン済みはDiscord名を自動入力（編集不可）、未ログインは手動入力（必須） |

### レスポンシブ対応方針

**レスポンシブ対応あり（PC・モバイル横・モバイル縦の3段階）。**

**レイアウト構成:**

| 環境 | ボタン操作エリア | メタデータエリア |
|---|---|---|
| PC（幅広） | 方向パッド＋ボタンパッド（横並び）| ボタン操作エリアの右カラム |
| モバイル横 | 方向パッド＋ボタンパッド（横並びを維持）| ボタン操作エリアの右または下 |
| モバイル縦 | 方向パッド＋ボタンパッド（横並びを維持）| ボタン操作エリアの下段 |

**方向パッドとボタンパッドは全環境で横並びを維持する。** モバイル縦でも2XKOMBOと同様に「方向パッド約3割・ボタンパッド約7割（3行×5列）」の構成で成立させる。詳細なボタン配置は `コンボ投稿ページ_入力UI.md` を参照。

**ブレイクポイント（実装時に調整可）:**
- `≥ 840px`: 2カラム（ボタン操作エリア左・メタデータ右）
- `< 840px`: 1カラム（ボタン操作エリア上・メタデータ下）

## Phase 2 設計

### ページ概要

不特定多数のユーザーがコンボを投稿できる公開フォームページ。投稿データは DB に保存され、新着コンボページ（`/recent/`）に反映される。

### 投稿フォームフィールド（Phase 2 追加・変更）

Phase 1 のフィールドに加えて以下を有効化:

| フィールド | Phase 2 での扱い |
|---|---|
| 認証 | Discord ログイン（任意）。未ログインでも投稿・閲覧は可能。My Combos のクラウド同期にはログイン必要 |
| 投稿者名 | ログイン済み: Discord ユーザー名を自動入力（**編集不可**・読み取り専用表示）。未ログイン: 手動入力（**必須**）。表示名の変更はプロフィールページで行う（未設計）|

### 投稿後の運用フロー（Phase 2 設計時に決定）

以下のいずれかを Phase 2 設計時に決定する:

- **即時反映:** 投稿データをそのまま DB に追加し `/recent/` に表示
- **審査制:** 投稿を一時保留し、管理者が承認後に公開

## 広告配置

### Phase 1

フォームページのため広告なし。

### Phase 2 以降

**フォームページのため原則広告なし。**

投稿フォームは「記入して送信する」という目的行動が明確なページ。広告による離脱・誤タップのリスクを避けるためフォーム内・近傍には広告を置かない。

#### 例外：送信完了後の導線エリア

投稿完了後に表示する「次のアクション（Discord でシェア / コンボを探す など）」の下に広告スロットを設ける余地はある。ただし実装コストに見合うかは Phase 2 設計時に判断する。

## 課題・検討事項（Phase 2 以降）

### スパム・不正投稿対策

- CAPTCHA の要否
- 投稿レート制限
- 審査フローの設計

### 投稿者の認証

- 匿名投稿を認めるか、Discord ログインを必須にするか
- Discord サーバーとの連携（投稿通知など）

### 入力表記のバリデーション（Phase 2 強化）

Phase 1 は管理者責任で入力するためバリデーション省略。Phase 2 では以下を追加:

- `()` ブロックの開閉対応チェック
- `duo_only` フラグとの整合性チェック（自動チェックも検討）
- 連続コネクタ・空トークン等の軽量警告
- パーサをフォーム側・ComboCard 表示側で共通化（実装メモ: `src/lib/combo-input-tokens.ts` 仮）

## ComboCard

### 構造

```
article.combo-card（border-radius: 8px, bg: surface, cursor: pointer）
├── .cc-title（コンボタイトル・省略可）  ← 最初に目に入る位置
├── .cc-header
│   ├── .cc-damage（ダメージ数値）    ← 左端・最初に目に入る
│   ├── .cc-difficulty（難易度星 + PULSEバッジ）
│   └── .cc-version（バージョン。game_version があれば表示）
├── .cc-badges（状況バッジ群）
├── .cc-inputs-wrap
│   ├── .cc-inputs.cc-inputs-short（短縮入力表記）
│   ├── .cc-inputs.cc-inputs-full（全入力表記, 初期非表示）
│   └── button.cc-expand-btn（「全表記」, 6手以上のみ・右端配置）
├── .cc-tags（追加要素タグ）           ← tags が空なら非表示
├── p.cc-notes（メモ）                ← notes がなければ非表示
└── .cc-footer
    ├── .cc-credit（投稿者名）
    └── .cc-actions
        ├── 練習用で開くリンク（.cc-detail-link）
        ├── リンクコピーボタン
        ├── ▶ 動画ボタン（.cc-video-btn）
        └── ブックマークボタン（disabled）
```

| バッジ               | 表示条件                                    | 色                              |
| ----------------- | --------------------------------------- | ------------------------------ |
| [Fuse名]限定         | `fuse !== 'No Fuse'`                    | `#f59e0b`（アンバー）                |

| DD · 2XA · FS     | `fuse === 'No Fuse'` かつ `tag === true`  | `var(--color-text-muted)`（グレー） |

| 位置（positionLabel） | 常時                                      | `#60a5fa`（青）                   |

| アシスト              | `combo.assist_used === true`            | `#fb923c`（オレンジ）                |
| デュオ               | `combo.duo_only === true`               | `#f472b6`（ピンク）                 |
| Pゲージ ×n           | `combo.ult_gauge_point > 0`             | `#38bdf8`（水色）                  |
| Aゲージ ×n           | `combo.ult_gauge_assist > 0`            | `#38bdf8`（水色）                  |

**Fuseバッジ派生ルール:**
- `[Fuse名]限定` バッジには実際のFuse名を表示（例: `Freestyle限定`、`Sidekick限定`）
- `DD · 2XA · FS` は `No Fuse` かつ交代あり = 交代可能フューズ全対応の意味。将来Fuseが増えた場合はこの列挙に追加する
- `フューズ不問` と `DD · 2XA · FS` はどちらも `No Fuse` コンボだが交代フラグで出し分ける

**Fuse略称（`DD · 2XA · FS` バッジ用）:**

| Fuse名 | 略称 | 交代可否 |
|---|---|---|
| Double Down | DD | ✅ |
| 2X Assist | 2XA | ✅ |
| Freestyle | FS | ✅ |
| Sidekick | SK | — |
| Juggernaut | Jug | — |

交代可能Fuseの定数はコード側で `TAG_CAPABLE_FUZES` として管理し、Fuse追加時はここを更新する。

### 入力表記（`.cc-inputs-wrap`）

- `bg: color-bg, border-radius: 6px, padding: 0.5rem 0.7rem`
- `code` タグ: monospace（Fira Code / Cascadia Code / JetBrains Mono）, font-size: 0.85rem, color: #fbbf24
- `white-space: nowrap`（横スクロール）

**短縮ルール（`shortenInputs()`）:**
- 5手以内（`>` 区切りで ≤5）: そのまま全表示
- 6手以上: `先頭2手 > ... > 末尾2手` で短縮 + 「全表記」ボタン（入力表記エリア右端に配置）
- ボタンクリックで短縮↔全表記トグル

**記法のレンダリング方針:**
- `mash.X` → そのままテキスト表示
- `duo.(X > Y)` 残キャラ入力ブロック → `duo.(` ～ `)` の範囲をサブカラー（例: #a78bfa 紫）でハイライト表示し、残キャラの入力であることを視覚的に区別する（デュオ専用コンボのみ出現）
- `(X)` オプション入力 → 括弧ごとやや薄いカラー（opacity 下げ等）で表示し、省略可能であることを示す
- `,` リンク → `>` と同様にそのままテキスト表示（見た目の区別はしない）

### フッター（`.cc-footer`）

**クレジット（`.cc-credit`）:**
- `.cc-submitter`: 投稿者名（0.65rem）

**アクションボタン（`.cc-actions`）:**

| ボタン | 動作 | 状態 |
|---|---|---|
| 練習用で開く | `<a href="/combo/{id}/">` で コンボ単体ページへ遷移 | 常時有効 |
| リンクコピー | `navigator.clipboard.writeText(url#combo-{id})` | 常時有効 |

**「練習用で開く」リンクの役割:**
- カード上の動画ボタン（▶）が「探すモードの軽いチェック」を担うのに対し、このリンクは「腰を据えて練習する」導線。コンボ単体ページ（`/combo/[id]/`）に遷移し、大画面動画 + 番号付き縦リスト入力表記のフォーカス表示に切替わる
- 設計詳細は [コンボ単体ページ.md](コンボ単体ページ.md) を参照

## フィルターバー（共通仕様）

### 常時表示行

| フィルター | UI | 備考 |
|---|---|---|
| 難易度 | チップ ★1〜★5 | ★1/★2 は PULSE バッジ付き |
| ポイントキャラ | チップ（すべて / [キャラA名] / [キャラB名]） | ページのキャラ名を動的表示。Juggernaut には表示しない。Sidekick では特に重要 |
| 始動 | `<select>`（すべて/地上/対空/崩し/タッグランチャー始動） | 単一選択。Sidekick/Juggernautでは `タッグランチャー始動` を除外 |
| 位置 | `<select>`（すべて / 位置不問 / 中央 / 端寄り / 端 / 端寄り(背) / 端(背)）計7値 | 単一選択。`position` は排他フィールドのため。「すべて」はフィルタなし、「位置不問」は `no_limit` コンボのみ表示 |
| 並び順 | `<select>` | 詳細は下記 |
| その他 ▼ | トグルボタン | Fuse別のみ（展開パネルあり）|

**チップスタイル:**
- 通常: `border: border, color: text-muted, bg: transparent`
- アクティブ: `border/color: accent, bg: accent 10% mix`
- hover: `border: accent, color: text`

**ソート選択肢（Fuse別 / Juggernaut で若干異なる）:**

デフォルトソートは `difficulty-asc`（難易度順・易しい順）。

| 値 | Fuse別ラベル | Juggernautラベル |
|---|---|---|
| `difficulty-asc` | 難易度順（易しい順）**【デフォルト】** | 難易度（低い順）**【デフォルト】** |
| `damage-desc` | ダメージ順 | ダメージ（高い順）|
| `difficulty-desc` | 難易度順（難しい順）| —（Juggernautにはなし）|
| `id-desc` | 新着順 | — |

## フィルタロジック（`pair-combo-page.ts`）

### マッチング条件（`matches()` 関数）

各フィルタは AND 結合。複数選択のチップは OR 結合。

```
難易度: state.difficulty に card.difficulty が含まれる（複数選択OR）
始動:   state.starter に card.starter が含まれる（複数選択OR）
位置:   state.position に card.position が含まれる（複数選択OR）
タグ:   state.tags のいずれかが card.tags に含まれる（OR）
アシスト: state.assist_used が true なら card.assistUsed も true
交代:   state.tag が true なら card.tag も true
デュオ:  state.duo_only が true なら card.duoOnly も true
Pゲージ: '0' → 0のみ, '1' → 1以上
Aゲージ: 同上
```

### ソート（`sortCards()` 関数）

| sort 値 | 第1キー | 第2キー |
|---|---|---|
| `difficulty-asc` | difficulty 昇順 | damage 降順 |
| `damage-desc` | damage 降順 | id 昇順 |
| `difficulty-desc` | difficulty 降順 | damage 降順 |

### URL クエリ同期

**Fuse別ページ専用。** Juggernaut / My Combos / 新着コンボには実装しない。

- `history.replaceState` でフィルタ状態をURLに反映
- デフォルト値はクエリに含めない（`difficulty-asc`・`page=1` は省略）
- Fuseタブのリンク href にも現在のクエリを引き継ぐ（タブ切替後もフィルタ維持）
- ページ読み込み時に `applyQueryState()` でURLから状態を復元

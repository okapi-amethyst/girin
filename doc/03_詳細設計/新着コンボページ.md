# 新着コンボページ 詳細設計

> 対象URL: `/recent/`
> 実装状況: Phase 1 — サンプルコンボ表示（静的）/ Phase 2 — DB連携
> 最終更新: 2026-04-12

---

## Phase 1 設計

### ページ概要

DB連携が必要なため Phase 1 では新着データを持てない。代替として Fuse別ページと同様のサンプルコンボ一覧を表示する静的ページとして実装する。
ナビゲーション（ヘッダー・各ページの導線）からのリンクは **Phase 1 では設置しない**。URL を直接入力したユーザーのみアクセスできる状態にとどめる。

### コンポーネント構成（Phase 1）

```
recent/index.astro
└── Base.astro
    └── .container.page
        ├── nav.breadcrumb（トップ / 新着コンボ）
        ├── header（Phase 2 告知バナー）
        └── section.combo-area
            ├── div.combo-list
            │   └── ComboCard × n（サンプルコンボ）
            └── nav.pagination
```

### Phase 1 の表示内容

- **コンボ一覧:** 既存のサンプルコンボを使用（Fuse別ページと同じデータ）
- **レイアウト:** Fuse別ページのコンボエリアと同じ構成（ComboCard + ページング）
- **フィルター:** なし
- **告知バナー:** 「新着コンボはコンボ投稿機能（準備中）が実装された後に表示されます。」程度のメッセージ

## Phase 2 設計

### ページ概要

投稿されたコンボを新着順（`created_at` 降順）で表示する。ペア・Fuse を横断して表示するため、ComboCard にペア情報（どのペアのコンボか）を付加する必要がある。

### コンポーネント構成（Phase 2）

```
recent/index.astro
└── Base.astro
    └── .container.page
        ├── nav.breadcrumb
        ├── header
        │   └── h1「新着コンボ」
        ├── div.filter-bar（簡易: 難易度 / キャラ絞り込み）
        └── section.combo-area
            ├── div.pair-group × n（ペアごと）
            │   ├── div.pair-group-header（キャラアイコン + ペア名）
            │   └── div.combo-list
            │       └── ComboCard × n
            └── nav.pagination
```

### Phase 2 の機能

- 新着コンボを `created_at` 降順でリスト表示
- ページ側でペアごとにコンボをグループ化し、グループヘッダー行（キャラアイコン + ペア名）で区切る。ComboCard 自体は拡張しない
- 難易度・キャラによる簡易フィルター
- ページング（件数は Fuse別ページと同じ 5件/ページ を基本とする）

## 広告配置（Phase 2〜）

### Phase 1

広告なし（noindex のサンプルデータページのため導入しない）。

### Phase 2 以降

Fuse別ページと近いレイアウト（コンボ一覧 + ページング）だが、フィルターはシンプルで動画パネルがない分だけ配置の自由度が高い。

#### 候補位置

| # | 位置 | セクション間の区切り |
|---|---|---|
| ① | `nav.pagination` の下（コンボリスト読み切り後） | ページング ↓ **[広告]** ↓ フッター |
| ② | `div.filter-bar` の上 ※慎重 | 見出し ↓ **[広告]** ↓ フィルターバー |

| 候補 | 推奨サイズ | 備考 |
|---|---|---|
| ① | `300×250` または Responsive | **最優先**。コンボを読み切った後の自然な位置 |
| ② | `320×50`（Mobile Banner）| フィルター前の薄い区切り。コンボ件数が増えてきた段階で検討 |

**①だけを先に導入し**、収益が安定してから②を検討する。

#### 避けるべき位置

- **ComboCard 間への混入**: コンテンツと誤認させる
- **フィルターバーの内部**: 操作の邪魔になる
- **告知バナーの近傍**: Phase 2 移行前後は告知バナーと広告が並ぶ可能性があり要注意

## 課題・検討事項

### Phase 1 のサンプルコンボ

- My Combos ページと同じサンプルデータを使うことになるため差別化が難しい
- Phase 1 では `/recent/` を完全に 404 のまま（ページ未生成）にしてもよいか再検討の余地がある

### 「新着」の定義

- DB 投稿日時を使う場合: `created_at` カラムが必要
- サイト追加日時を使う場合: 運営が手動で `added_at` を管理
- どちらにするかは Phase 2 のデータ設計時に決定

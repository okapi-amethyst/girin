# キャラ単体ページ 詳細設計

> 対象URL: `/[char]/`
> 実装ファイル: `src/pages/[slug]/index.astro`
> 最終更新: 2026-04-08

---

## ページ概要

特定キャラクターの相方を選ぶ起点ページ。
相方グリッドからペアトップページへ、JUGGERNAUTリンクからJuggernautページへ遷移できる。

---

## コンポーネント構成

```
[slug]/index.astro
└── Base.astro（ヘッダー・フッター）
    └── .container.page（padding-top: 1.5rem）
        ├── nav.breadcrumb            ← パンくず
        ├── header.char-header        ← キャラヘッダー
        ├── section.partner-section   ← パートナー選択グリッド
        │   ├── div.section-header（flex row）
        │   │   ├── h2「パートナーチャンピオンを選択」
        │   │   └── a.juggernaut-link（JUGGERNAUTリンク）← 見出し右隣に配置
        │   └── .partner-grid
        │       └── a.partner-card × (全キャラ数 - 1)
        └── div.back-to-top           ← 「← トップに戻る」テキストリンク
```

---

## 各コンポーネント詳細

### パンくずナビ（`nav.breadcrumb`）

```
トップ / キャラ名
```

- font-size: 0.85rem, color: text-muted
- トップはリンク、キャラ名は現在地（非リンク）

---

### キャラヘッダー（`header.char-header`）

| 要素 | 内容 |
|---|---|
| `.char-icon-portrait` | キャラ画像（56px 丸型, background-image クロップ）|
| フォールバック | 画像なし時: 頭文字 56px 丸 |
| `h1` | キャラ英語名（1.6rem, font-weight: 800）|
| `.char-name-ja` | キャラ日本語名（0.9rem, color: text-muted）|

---

### パートナー選択グリッド（`section.partner-section`）

#### セクションヘッダー（`div.section-header`）

`display: flex; align-items: center; gap: 0.75rem` の横並び行。

| 要素 | 内容 |
|---|---|
| `h2.section-title` | 「パートナーチャンピオンを選択」（1rem, font-weight: 700）|
| `a.juggernaut-link` | JUGGERNAUTリンク（見出し直右に配置） |

**JUGGERNAUTリンク（`a.juggernaut-link`）:**
- インラインフレックスカード（bg: surface, border: border, border-radius: 8px）
- Fuseアイコン: `public/images/fuses/Juggernaut_fuse_360.png`（28px×28px, border-radius: 4px）
- 名前: 0.8rem, font-weight: 700 / サブ: 0.65rem, color: text-muted
- ホバー: border-color → accent, color → text
- リンク先: `/[char.slug]/juggernaut/`
- **配置方針:** `section-header` 内の見出し直後に隣接。左→右の読み順で自然に目に入る位置。`space-between` は使わない（右端は離れすぎ）

#### グリッド本体

- 対象キャラ以外の全キャラを表示（自分自身は除外）
- グリッドカラム数（レスポンシブ）:

| ブレークポイント | カラム数 | gap |
|---|---|---|
| ～639px | 3列 | 0.5rem |
| 640px～ | 4列 | 0.75rem |
| 960px～ | **6列** | 0.75rem |

> 13キャラ構成では自身を除く12パートナーがちょうど 2行×6列に収まる。

**パートナーカード（`a.partner-card`）:**

| 要素 | 内容 |
|---|---|
| `.partner-portrait` | キャラ画像（aspect-ratio: 6/5, クロップ）|
| フォールバック | 頭文字 |
| `.partner-label` | キャラ名（0.8rem, overflow: ellipsis）|
| ホバー | border-color → accent |

**リンク先 URL のペア順序ルール:**
- `characters` 配列のインデックスが小さい方が `char1`（CHARACTER_MASTERの `id` 順）
- 例: Ahri(index 0) + Yasuo(index 11) → `/ahri/yasuo/`
- 逆順パターン（`/yasuo/ahri/`）は生成せず、`/ahri/yasuo/` にリダイレクト（現状リダイレクト未実装）

---

### トップ戻りリンク（`div.back-to-top`）

```
← トップに戻る
```

- font-size: 0.85rem, color: text-muted / ホバー: color → text
- リンク先: `/`（BASE_URL）
- margin-top: 2rem
- **設計方針:** ページ下部まで読んだユーザーへの帰り道として設置。機能導線カード（新着・My Combos・コンボ投稿）は不要なため廃止

## 広告配置（要検討 / Phase 2〜）

> トップページより 1 ページ深い「遷移中」のページ。コンテンツ量が少なく、広告が占める割合が大きくなりやすい点に注意。

### 候補位置

| # | 位置 | セクション間の区切り |
|---|---|---|
| ① | `header.char-header` と `section.partner-section` の間 | キャラヘッダー ↓ **[広告]** ↓「パートナーチャンピオンを選択」グリッド |
| ② | `.partner-grid` の下（`div.back-to-top` の上） | パートナーグリッド ↓ **[広告]** ↓「← トップに戻る」 |

### モバイル（〜639px）

| 候補 | 推奨サイズ | 備考 |
|---|---|---|
| ① | `320×50` または Responsive | ヘッダーとグリッドの区切りとして機能。ページが短いため目立ちすぎに注意 |
| ② | `300×250` または Responsive | グリッド下・戻りリンクの前。こちらの方が UX 影響が少ない |

- ページ全体のコンテンツ量が少ないため **①②同時設置は避ける**。②を優先する。

### PC（960px〜）

| 候補 | 推奨サイズ | 備考 |
|---|---|---|
| ① | `728×90` または Responsive | — |
| ② | `300×250` または Responsive | — |

### 避けるべき位置

- **相方グリッドの途中への混入**: キャラカードと誤認されるため不可
- **パンくずナビの上**: ページ最上部に広告が来るとユーザーの離脱を招く

## 課題・検討事項

### ページの情報量

- 現状は相方グリッドと Juggernaut リンクのみで、キャラ自体の情報が一切ない
- キャラの特性説明（1〜2行のテキスト）や、おすすめペアのハイライトを追加する案がある

- Astro の `redirect` または `getStaticPaths` での対応が必要

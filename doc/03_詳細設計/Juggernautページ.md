# Juggernautページ 詳細設計

> 対象URL: `/[char]/juggernaut/`
> 実装ファイル: `src/pages/[slug]/juggernaut.astro`
> 最終更新: 2026-04-08

---

## ページ概要

ソロFuse「JUGGERNAUT」のコンボ一覧ページ。
アシスト・交代・DUOに関するフィルタを持たない簡易版フィルターバーを使用する点が Fuse別ページとの主な違い。

---

## コンポーネント構成

```
[slug]/juggernaut.astro
└── Base.astro（ヘッダー・フッター）
    └── .container.page（padding-top: 1.5rem）
        ├── nav.breadcrumb            ← パンくず（3段）
        ├── header.char-header        ← キャラヘッダー
        ├── div.filter-bar            ← 簡易フィルターバー（展開パネルなし）
        ├── section.combo-area        ← コンボ一覧（現状は空件メッセージのみ）
        └── div.back-to-top           ← 「← トップに戻る」テキストリンク
```

## 各コンポーネント詳細

### パンくずナビ（`nav.breadcrumb`）

```
トップ / キャラ名 / JUGGERNAUT
```

- トップ・キャラ名はリンク、JUGGERNAUT は現在地（非リンク）
- Fuse別ページとの違い: **ペアページを経由する階層になっていない**（ソロFuseのためキャラ配下）

---

### キャラヘッダー（`header.char-header`）

| 要素 | 内容 |
|---|---|
| `.char-icon-portrait` | キャラ画像（56px 丸型）|
| フォールバック | 頭文字 |
| `h1` | キャラ英語名（1.6rem, font-weight: 800）|
| `.fuse-label` | `JUGGERNAUT`（accent色, 0.85rem, uppercase, letter-spacing: 0.05em）|

- キャラ単体ページのヘッダーと同構造。`fuse-label` が追加されている点のみ異なる

---

### フィルターバー（`div.filter-bar`）

Fuse別ページの常時表示行のみ。展開パネルなし。

| フィルター | UI |
|---|---|
| 難易度 | チップ（★1〜★5, PULSE バッジ付き）|
| 始動 | チップ（地上 / 対空 / CH / 投げ / 崩し）|
| 位置 | チップ（中央 / 端）|
| 並び順 | `<select>`（実用度 / 難易度（低い順）/ ダメージ（高い順））|

**Fuse別ページとの差分:**
- 展開パネル（アシスト・交代・DUO・ゲージ・タグ）なし
- Headroom ナビなし
- 「その他 ▼」ボタンなし
- ソート選択肢のラベルが若干異なる（「難易度（低い順）」など）

**現状の制限:**
- フィルターの状態が URL に反映されない（チップのオン/オフはクライアント側の見た目のみ、実際のフィルタリングは未実装）
- コンボデータが存在しないため動作確認不可

---

### コンボエリア（`section.combo-area`）

現状は空件メッセージのみ表示:

```
コンボデータ準備中
JUGGERNAUTコンボが登録されると、ここに表示されます
```

- Fuse別ページの空件メッセージ（Discord 導線付き）とは異なるメッセージ
- 実データ投入後は ComboCard の一覧 + ページングが表示される予定

---

### トップ戻りリンク（`div.back-to-top`）

```
← トップに戻る
```

- font-size: 0.85rem, color: text-muted / ホバー: color → text
- リンク先: `/`（BASE_URL）
- margin-top: 2rem
- **設計方針:** コンボを読み終えたユーザーへの帰り道。My Combos / Submit Combo 等の機能導線はこのページ文脈では不要なため廃止

## 広告配置（要検討 / Phase 2〜）

> Fuse別ページと同様の方針。ただしページ構成がシンプル（Headroom ナビ・2カラムなし）のため、配置の難易度は低い。

### 候補位置

| # | 位置 | セクション間の区切り |
|---|---|---|
| ① | `section.combo-area` の下（ページング下またはコンボリスト後） | コンボリスト/空件メッセージ ↓ **[広告]** ↓「← トップに戻る」 |
| ② | `div.filter-bar` の上 ※慎重 | キャラヘッダー ↓ **[広告]** ↓ フィルターバー |

### モバイル（〜639px）

| 候補 | 推奨サイズ | 備考 |
|---|---|---|
| ① | `300×250` または Responsive | **最優先**。Fuse別ページと同方針。コンボを読み切った後の自然な着地点 |
| ② | `320×50` または Responsive | フィルター前に薄く挟む案。Juggernaut ページはシンプルなため許容されやすいが、コンボ件数が少ない場合ページの大半が広告になるリスクあり |

### PC（960px〜）

> 現状は縦並び（モバイルと同構成）のため、モバイルと同じ位置・サイズで対応。
> 将来的に PC 専用 2 カラムレイアウトを採用する場合は Fuse別ページに準じて再検討する。

| 候補 | 推奨サイズ | 備考 |
|---|---|---|
| ① | `728×90` または Responsive | コンテナ中央寄せ |

### 避けるべき位置

- **フィルターバーとコンボカードの間**: 「フィルターを操作したらすぐ広告」は不快感が高い
- **ページ最上部（パンくずの上）**: ページ表示直後の広告は直帰率を上げる

## 課題・検討事項

### フィルタ・URLクエリの未実装

- チップのトグルは見た目だけ動くが、コンボカードの絞り込みは未実装
- 実データ投入のタイミングで Fuse別ページと同様の `initPairComboPage()` 相当のロジックを実装する必要がある

### 空件メッセージのトーン

- 「コンボデータ準備中」は内部向けのメッセージで、一般ユーザーには「まだコンボがありません + Discord 投稿導線」に統一したい

### Juggernautページへの導線

- キャラ単体ページの Juggernaut リンクからのみ到達できる
- Fuse別ページの Fuse タブナビには Juggernaut が含まれていない（設計上はソロFuseのためペア配下に置かない方針）
- ペアページから Juggernaut ページへの導線が現状ない（必要かどうかは要検討）

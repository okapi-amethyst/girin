# ナビ・ヘッダー・フッター 詳細設計

> 実装ファイル: `src/layouts/Base.astro`
> 最終更新: 2026-04-05

---

## 全体構造

```
Base.astro
├── <head>                    ← メタ情報・OGP・テーマ初期化スクリプト
├── <body>
│   ├── header.site-header    ← グローバルヘッダー
│   ├── main > <slot />       ← 各ページのコンテンツ
│   └── footer.site-footer    ← グローバルフッター
└── <script>                  ← テーマ切替・シェアボタン
```

## `<head>`

| 要素 | 内容 |
|---|---|
| charset | UTF-8 |
| favicon | `/favicon.svg` |
| viewport | width=device-width, initial-scale=1 |
| description | ページ固有 or デフォルト `"2XKO GIRIN — これだけ覚えたら対戦行ってOK"` |
| og:title | ページタイトル |
| og:description | ページ説明 |
| og:image | ページ固有 or デフォルト `/og-default.png` |
| og:type | `website` |
| og:url | 現在の URL（`Astro.url`）|
| twitter:card | `summary_large_image` |
| title | ページタイトル |

**テーマ初期化（`is:inline` スクリプト）:**
- `localStorage.getItem('theme')` があれば `data-theme` に適用
- なければ `prefers-color-scheme: light` に応じて自動設定
- ページ描画前に実行（FOUC防止）

## ヘッダー（`header.site-header`）

```
[ GIRIN ]  [ 2XKO ]                    （右寄せ）  [ ~~新着~~ ]  [ ~~My Combos~~ ]  [ ~~コンボ投稿~~ ]  [ Search(disabled) ]  [ ☀/☾ ]
```

### レイアウト

- ロゴ・2XKO バッジが左端
- `.header-nav-start`（新着）に `margin-left: auto` を設定することで、以降の要素が右寄せになる

### 要素一覧

| 要素 | クラス | 内容 | 状態 |
|---|---|---|---|
| ロゴ | `.site-logo` | GIRIN テキスト | トップへリンク |
| ゲームバッジ | `.site-badge` | `2XKO` | accent背景, 非リンク |
| 新着 | `span.header-link.is-disabled.header-nav-start` | 取り消し線テキスト | **リンクなし**（`<span><s>`）。Phase 2 で `<a>` に変更 |
| My Combos | `span.header-link.is-disabled` | 取り消し線テキスト | 同上 |
| コンボ投稿 | `span.header-link.is-disabled` | 取り消し線テキスト | 同上 |
| Search | `.header-icon-btn[disabled]` | 将来の検索用プレースホルダー | disabled（opacity 0.45）|
| テーマ切替 | `.theme-toggle` | ☀（ライト時）/ ☾（ダーク時）| 右端 |

**Phase 1 の方針:** 新着・My Combos・コンボ投稿は `<span><s>` による取り消し線テキストで配置。Coming Soon バッジは使用しない（スペース節約・見た目検証の邪魔にならないため）。Phase 2 でページ実装時に `<a href="...">` に差し替える。

**Share ボタンは削除済み（Phase 1 では不要）。** 必要になった場合は各ページ内に個別ボタンとして配置する想定。

### スタイル

- `border-bottom: 1px solid var(--color-border)`, `padding: 0.75rem 0`
- コンテナ内: `display: flex, align-items: center, gap: 0.75rem, flex-wrap: wrap`
- `.site-logo`: font-size 1.1rem, font-weight 700
- `.site-badge`: accent 背景, `#0a0a0f` 文字, 0.7rem uppercase

### Share ボタンの動作

1. `navigator.clipboard.writeText(window.location.href)` でURL全体をコピー
2. 成功: 「コピーしました」を 1.8 秒表示
3. 失敗: 「コピーに失敗しました」を表示

### テーマ切替の動作

- クリックで `dark` ↔ `light` をトグル
- `data-theme` 属性を `<html>` に付与 + `localStorage` に保存
- CSS: `dark` 時は `.theme-icon-light` を非表示、`light` 時は `.theme-icon-dark` を非表示

## パンくずナビ（各ページ実装）

Base.astro には含まれず各ページで独自実装。パターンは以下の3種:

| ページ | パンくず |
|---|---|
| キャラ単体 | トップ / キャラ名（現在地）|
| Juggernaut | トップ / キャラ名 / JUGGERNAUT（現在地）|
| Fuse別・ペアトップ | トップ のみ（課題: 階層が反映されていない）|

## Headroom ナビ（Fuse別ページのみ）

詳細は [Fuse別ページ.md](./Fuse別ページ.md) を参照。

`position: fixed; top: 0; z-index: 200` で、フィルターバーがスクロールアウトしたときに表示。動画パネルの `top` 値と連動。

## 課題・検討事項

### ヘッダーのレスポンシブ

- 現状 `flex-wrap: wrap` で対応しているが、スマホ幅でアイテムが折り返す可能性がある
- 将来の機能追加でアイテムが増えた場合はハンバーガーメニューへの格納を検討（設計書記載済み）

### Share ボタンのテキスト

- 現状「Share」テキストのみ。アイコン化する案がある

### Search プレースホルダー

- 将来のサイト内検索用に場所を確保しているだけで機能なし
- Phase 1 は disabled のまま

### パンくずの不統一

- Fuse別ページのパンくずが「トップ」だけで、ペア階層が表示されない
- `トップ / キャラA名 / キャラA × キャラB / FUSE名` に整備する必要がある

### フッターのリンク先

- Discord URL・X URL が仮の `discord.gg/` / `x.com/` のままでプレースホルダー状態
- 実際のサーバー・アカウント URL に差し替えが必要

### Discord Bot 招待リンク・お問い合わせ

- 設計書（サイト構造設計.md）にはフッターに Discord Bot 招待リンクを含める記載があるが現状未実装

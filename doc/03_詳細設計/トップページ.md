# トップページ 詳細設計

> 対象URL: `/`
> 実装ファイル: `src/pages/index.astro`
> 最終更新: 2026-04-08

---

## ページ概要

サイトへの入口。キャラクターを選択してコンボを探す起点となるページ。
また新着コンボ・My Combos・コンボ投稿への機能導線カードも設置する。

---

## コンポーネント構成

```
index.astro
└── Base.astro（ヘッダー・フッター）
    ├── section.hero              ← キャッチコピー
    ├── section.nav-links         ← 機能導線カード
    │   └── .nav-links-row（flex, 中央寄せ）
    │       ├── .nav-link-card（新着コンボ）
    │       ├── .nav-link-card（My Combos）
    │       └── .nav-link-card（コンボ投稿）
    └── section.characters        ← キャラ選択グリッド
        ├── h2.char-grid-title（「コンボを調べたいチャンピオンを選択」）
        └── .char-grid
            └── a.char-card × n
                ├── .char-portrait（キャラ画像）
                └── .char-label（キャラ名）
```

### キャラ選択グリッド（`section.characters`）

グリッド直上に `h2.char-grid-title`「コンボを調べたいチャンピオンを選択」を配置（1rem, font-weight: 700）。

**グリッドカラム数（レスポンシブ）:**

| ブレークポイント | カラム数 | gap |
|---|---|---|
| ～639px | 3列 | 0.5rem |
| 640px～ | 4列 | 0.75rem |
| 960px～ | **6列** | 0.75rem |

> PC 6列・13キャラの場合: 2行（6+6）+ 最終行1枚。14キャラ追加後は2行（6+6）+ 最終行2枚。

**キャラカード（`a.char-card`）:**

| 要素 | 内容 |
|---|---|
| `.char-portrait` | キャラ画像（aspect-ratio: 6/5, background-image でクロップ）|
| フォールバック | 画像なし時: 名前の頭文字 |
| `.char-label` | キャラ名（0.8rem, overflow: ellipsis）|
| ホバー | border-color → accent |

- リンク先: `/[char.slug]/`
- 画像スケール: JS で `data-size` / `data-pos-x` / `data-pos-y` を元に `background-size/position` を動的計算（WIKI_REF: 180px 基準）

## 広告配置（要検討 / Phase 2〜）

> AdSense 等のバナー広告を Phase 2 から導入する想定。以下は「邪魔にならない位置」の候補案。
> 実装時は実際のレイアウトと照らし合わせて最終決定すること。

### PC サイドバー広告（実装済み・Phase 2 で有効化）

viewport 1360px 以上で、コンテンツ（960px）の左右両外側に固定サイドバー広告スロットを配置済み。

```
[左広告 160px] ← 36px gap → [コンテンツ 960px] ← 36px gap → [右広告 160px]
                     合計 min-width: 1360px で表示
```

- 実装: `Base.astro` に `<aside class="ad-sidebar-left/right">` を `position: fixed` で配置
- Phase 1 では `.ad-placeholder`（破線枠+「広告 160×600 Phase 2〜」）を表示
- Phase 2 移行時: `.ad-placeholder` を AdSense タグに差し替えるだけ
- z-index: 10（headroom-nav: 200 より低い）
- **左サイドバーは将来のナビゲーション/メニュー転用も想定**

### コンテンツ内広告候補位置（モバイル・狭い PC）

| # | 位置 | セクション間の区切り |
|---|---|---|
| ① | `section.nav-links` と `section.characters` の間 | 機能導線カード ↓ **[広告]** ↓ キャラグリッド |
| ② | キャラグリッドの下（ページ末尾） | キャラグリッド ↓ **[広告]** ↓ フッター |

### モバイル（〜639px）

| 候補 | 推奨サイズ | 備考 |
|---|---|---|
| ① | `320×50`（Mobile Banner）または Responsive | キャラグリッドの直前に水平線として自然に入る |
| ② | `300×250`（Medium Rectangle）または Responsive | スクロール終端の自然な着地点。最も UX への影響が少ない |

- **①と②の両方を同時に設置しない。**まず②から試験して CTR を確認する。

### PC（960px〜・サイドバー非表示幅）

| 候補 | 推奨サイズ | 備考 |
|---|---|---|
| ① | `728×90`（Leaderboard）または Responsive | コンテナ幅 960px に収まる水平バナー |
| ② | `728×90` または `300×250` | コンテナ中央寄せ |

### 避けるべき位置

- **ヒーローエリア内・前後**: ファーストビューで広告が目立つとブランドイメージを損なう
- **キャラグリッド内に混入**: キャラカードと見分けがつかず誤タップを誘発する

## ページ概要

**「このコンボを練習する」ための専用ページ。** ユーザーが一覧ページから探して決めた後の着地点であり、動画と全入力表記を腰を据えて見るためのフォーカス表示。

Fuse別ページの動画パネルが「探すモードの補助表示」なのに対し、このページは **練習モードが主役**。1ページ1コンボで、動画・全入力表記・詳細情報を最大限の画面領域で表示する。

### 位置づけの対比

| ページ | 役割 | 動画の扱い |
|---|---|---|
| Fuse別ページ | 探す（フィルタ・一覧） | 補助表示（sticky の軽いパネル） |
| **コンボ単体ページ** | 練習する（1コンボに集中） | **主役（ページのメインコンテンツ）** |

### 想定ユーザー動線

```
A. Fuse別ページで探して決める
    → ComboCard の「練習用で開く」リンク
    → /combo/[id]/

B. Discord で共有されたリンクを開く
    → Bot の /combo [ID] 埋め込み → Web 詳細リンク
    → /combo/[id]/

C. 友人から SNS / Discord で直接リンクが送られてくる
    → OGP プレビュー
    → /combo/[id]/

D. My Combos（Phase 2）からブックマーク済みコンボを開く
    → /combo/[id]/
```

## 各コンポーネント詳細

### パンくず（`nav.breadcrumb`）

パンくずの canonical 親は **ペアトップページ** とする。Fuse別ページは URL 上のカテゴリ階層だが、コンボ単体ページはどの Fuse から来ても同じ `/combo/[id]/` なので、ペアトップを正規の親とする。

```
トップ > アーリ×ヤスオ > #42
  ↓          ↓              ↓
  /    /ahri/yasuo/     現在地
```

- Juggernaut の場合は `トップ > キャラ名 > JUGGERNAUT > #42`（親がキャラ単体ページ）
- No Fuse（ペアトップのコンボ）は `トップ > アーリ×ヤスオ > #42`

#### Fuse別ページからの場合の戻り導線

パンくずはどこから来ても canonical 表示で固定するが、**Fuse別ページのリンクから遷移した場合** は URL の `?from=fuse` クエリパラメータを付けてリンクを渡し、クライアント側でパンくずにFuse段を追加表示する。

```
Fuse別ページから:
  トップ > アーリ×ヤスオ > Double Down > #42
                             ↑ ?from=fuse があるときのみ表示

直接アクセス・Discord・SNS から:
  トップ > アーリ×ヤスオ > #42
```

**実装方針:**
- `ComboCard` の `.cc-detail-link` は `/combo/42/?from=fuse` として生成（Fuse別ページ・Juggernautページから開く場合）
- 新着コンボ・My Combos・トップ「最近追加」等からのリンクは `?from=` なしで生成 → canonical 表示固定
- クライアント JS で `URLSearchParams` を読み取り、`from=fuse` があればパンくずに Fuse 段を挿入。それ以外はすべて canonical 表示
- SSG ページなので SSR は不要。クライアント側の条件分岐のみ

### 練習ステージ（`section.practice-stage`）— ページ主役

動画を見ながら全入力表記を番号で参照しながら手を動かす領域。**画面の中で最も目立つ位置・サイズを占める。**

#### 入力表記の表示形式

動画を見ながら入力表記を照合する際、横書き（`A > B > C`）より番号付き縦リストのほうが「今何手目か」を番号で直接参照できる。モバイル横は画面高さの制約が大きいため横書きを維持し、縦表示したい場合はデバイスを縦にしてもらう割り切りとする。

| レイアウト | 形式 | 理由 |
|---|---|---|
| **PC（≥ 840px）** | 番号付き縦リスト・動画の右 | 攻撃対象が動画右寄りにいるため視線移動が最小 |
| **モバイル縦（portrait）** | 番号付き縦リスト・動画下の右カラム | 同原則を維持。左エリアはコンボ情報表示領域 |
| **モバイル横（landscape）** | 横書き（短縮なし） | 画面高さ不足。動画を大きく見ることを優先 |

**縦リストの仕様:**
- 各手を1行で表示し行頭に番号を付ける（例: `4. j.H`）
- 動画の高さに揃えた固定高さ・縦スクロール可（20手超のコンボに対応）
- monospace フォント
- 入力の色は `#fbbf24`（ComboCard の入力表記と統一）

---

#### PC レイアウト（≥ 840px）

動画がページ中央で大きく表示され、右に縦リストが並ぶ。

```
[ .practice-stage（横並び・最大幅フルに近い） ]
  ├── .ps-video（16:9 iframe・flex-grow: 1）
  └── .ps-inputs-vertical（固定幅 260px 程度・動画と同じ高さ・縦スクロール）
      1. M
      2. H
      3. 2H
      4. jc
      5. j.M
      ...
```

- 動画はページ幅の許す限り大きく（例: 最大 720px 幅）
- 入力表記カラムは動画の右に隣接
- スクロールで動画が画面外に出ると、動画と入力カラムが揃って sticky 追従するかは要検討（常時表示かスクロール可かは課題）

#### モバイル縦（portrait）レイアウト

```
[ .practice-stage（縦並び） ]
  ├── .ps-video（16:9 iframe・width: 100%）
  └── .ps-bottom（横2カラム）
      ├── .ps-info-left（左エリア: 難易度・ダメージなどの簡易情報）
      │   └── 星表示 / ダメージ / #ID
      └── .ps-inputs-vertical（右カラム: 番号付き縦リスト・固定高さ・スクロール）
          1. M
          2. H
          3. 2H
          ...
```

- 左エリア: コンボの基本情報（難易度・ダメージ）を表示。
- 右カラム: 縦リスト。スクロールはこのカラム内で完結

#### モバイル横（landscape）レイアウト

```
[ .practice-stage（縦並び） ]
  ├── .ps-video（16:9 iframe・width: 100%、高さ優先）
  └── .ps-inputs-horizontal
      └── <code>入力表記（横書き全表記・monospace・折り返しあり）</code>
```

- 動画を最大限高く表示する。入力表記は動画の下に折り返し表示

練習ステージの下にコンボの詳細情報をまとめて表示。

| 要素 | 内容 |

## 挙動

| アクション | 結果 |
|---|---|
| ページ初期表示 | 動画 iframe は**自動ロードしない**。ポスター画像 + ▶ オーバーレイでユーザー操作を待つ |
| ▶ オーバーレイクリック | YouTube iframe の src をセットし再生開始 |
| パンくずクリック | Fuse別ページ等へ遷移 |
| リンクコピー | Toast で「コピーしました」|

- **自動再生しない理由:** モバイルでの想定外の通信負荷・共有リンク経由で開いた際の驚き回避。ユーザーの意思で再生開始する
- Esc キーの役割は Fuse別ページと違い「パネルを閉じる」ではなく、ブラウザ戻る or 何もしない

## 広告配置（要検討 / Phase 2〜）

> **このページは Fuse別ページと並んでユーザーの滞在時間が長いコア体験ページ。** 練習のために繰り返しスクロールする用途を考えると、動画・入力表記・コンボ情報の周辺に広告を置くことは大きな UX 破壊になる。

### 禁止エリア（絶対に広告を置かない）

| エリア | 理由 |
|---|---|
| `section.practice-stage` の内部・前後 | ページの主役。動画と入力表記への集中を妨げる広告は許容できない |
| `header.combo-header` の上 | ページ表示直後に広告が来るとユーザーの離脱を招く |
| `section.combo-info` の内部 | バッジ・タグ・メモの読み取りを分断する |
| `section.share-actions` の内部 | ボタン操作と誤認しやすい |

### ページのコンテンツフロー

```
パンくず
  ↓
combo-header（ペアアイコン + タイトル）
  ↓
practice-stage（動画 + 入力表記）← 禁止
  ↓
combo-info（メタ情報・バッジ・タグ・ノート）← 禁止
  ↓
--- ① 広告候補（練習・情報確認フェーズの終わり）---
  ↓
share-actions（共有ボタン群）
  ↓
--- ② 広告候補（ページ読み切り後）---
  ↓
フッター
```

### 候補位置

| # | 位置 | セクション間の区切り |
|---|---|---|
| ① | `combo-info` と `share-actions` の間 | コンボ情報を読み終えた自然な区切り |
| ② | `share-actions` の下（フッター直前） | ページを最後まで読んだユーザーのみに表示。UX 影響が最小 |

### モバイル（〜839px）

| 候補 | 推奨サイズ | 備考 |
|---|---|---|
| ② | `300×250` または Responsive | **最優先**。ページ読み切りタイミングで最も自然 |
| ① | `300×250` または Responsive | ②が安定したら追加検討 |

- **②だけを先に導入**し、収益が安定してから①の検討を推奨

### PC（840px〜）

**PC 1360px 以上の場合:** Base.astro のサイドバー広告（160×600）がすでに表示されるため、コンテンツ内広告は非表示にする（トップページ・キャラ単体ページと同じ方針）。

**PC 840px〜1359px（サイドバーなし）の場合:**

| 候補 | 推奨サイズ | 備考 |
|---|---|---|
| ② | `728×90`（Leaderboard）または Responsive | コンテナ中央寄せ。フッター直前 |
| ① | `728×90` または Responsive | ③が安定したら追加検討 |

### コンボ単体ページ特有の注意点

- **ユーザーはスクロールを繰り返す:** 動画を止めてコンボ情報を確認し、また動画に戻る往復が起きる。コンテンツ内広告はスクロール中に何度も目に入るため、①・②に配置する場合は特に慎重に
- **Discord・SNS 経由の初訪問が多い:** 「コンボを共有した側に招待された」ユーザーが多く、GIRINへの印象が薄い段階。最初に広告が目立つと離脱率を上げる
